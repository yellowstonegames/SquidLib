010/**
011 * This class provides methods for calculating Field of View in grids. Field of
012 * View (FOV) algorithms determine how much area surrounding a point can be
013 * seen. They return a two dimensional array of doubles, representing the amount
014 * of view (typically sight, but perhaps sound, smell, etc.) which the origin
015 * has of each cell.
016 * <br>
017 * The input resistanceMap is considered the percent of opacity. This resistance
018 * is on top of the resistance applied from the light spreading out. You can
019 * obtain a resistance map easily with the DungeonUtility.generateResistances()
020 * method, which uses defaults for common chars used in SquidLib, but you may
021 * also want to create a resistance map manually if a given char means something
022 * very different in your game. This is easy enough to do by looping over all the
023 * x,y positions in your char[][] map and running a switch statement on each char,
024 * assigning a double to the same x,y position in a double[][]. The value should
025 * be between 0.0 (unblocked) for things light passes through, 1.0 (blocked) for
026 * things light can't pass at all, and possibly other values if you have
027 * translucent obstacles.
028 * <br>
029 * The returned light map is considered the percent of light in the cells.
030 * <br>
031 * Not all implementations are required to provide percentage levels of light.
032 * In such cases the returned values will be 0 for no light and 1.0 for fully
033 * lit. Implementations that return this way note so in their documentation.
034 * Currently, all implementations do provide percentage levels.
035 * <br>
036 * All solvers perform bounds checking so solid borders in the map are not
037 * required.
038 * <br>
039 * Static methods are provided to add together FOV maps in the simple way
040 * (disregarding visibility of distant FOV from a given cell), or the more
041 * practical way for roguelikes (where a cell needs to be within line-of-sight
042 * in the first place for a distant light to illuminate it). The second method
043 * relies on an LOS map, which is essentially the same as a very-high-radius
044 * FOV map and can be easily obtained with calculateLOSMap().
045 * <br>
046 * If you want to iterate through cells that are visible in a double[][] returned
047 * by FOV, you can pass that double[][] to the constructor for Region, and you
048 * can use the Region as a reliably-ordered List of Coord (among other things).
049 * The order Region iterates in is somewhat strange, and doesn't, for example,
050 * start at the center of an FOV map, but it will be the same every time you
051 * create a Region with the same FOV map (or the same visible Coords).
052 *
053 * @author Eben Howard - http://squidpony.com - howard@squidpony.com
054 */
055public class FOV implements Serializable {

158    /**
159     * Calculates the Field Of View for the provided map from the given x, y
160     * coordinates. Returns a light map where the values represent a percentage
161     * of fully lit.
162     *
163     * The starting point for the calculation is considered to be at the center
164     * of the origin cell. Radius determinations are determined by the provided
165     * RadiusStrategy.
166     *
167     * @param resistanceMap the grid of cells to calculate on; the kind made by DungeonUtility.generateResistances()
168     * @param startX the horizontal component of the starting location
169     * @param startY the vertical component of the starting location
170     * @param radius the distance the light will extend to
171     * @param radiusTechnique provides a means to calculate the radius as desired
172     * @return the computed light grid
173     */
174    public double[][] calculateFOV(double[][] resistanceMap, int startX, int startY, double radius, Radius radiusTechnique) {
175        double rad = Math.max(1, radius);
176        double decay = 1.0 / rad;
177
178        int width = resistanceMap.length;
179        int height = resistanceMap[0].length;
180
181        initializeLightMap(width, height);
182        light[startX][startY] = 1;//make the starting space full power
183
184        switch (type) {
185            case RIPPLE:
186            case RIPPLE_LOOSE:
187            case RIPPLE_TIGHT:
188            case RIPPLE_VERY_LOOSE:
189                initializeNearLight(width, height);
190                doRippleFOV(light, rippleValue(type), startX, startY, startX, startY, decay, rad, resistanceMap, nearLight, radiusTechnique);
191                break;
192            case SHADOW:
193                // hotfix for too large radius -> set to longest possible straight-line Manhattan distance instead
194                // does not cause problems with brightness falloff because shadowcasting is on/off
195
196                // this should be fixed now, sorta. the distance is checked in the method this calls, so it doesn't ever
197                // run through more than (width + height) iterations of the radius-related loop (which seemed to be the
198                // only problem, running through billions of iterations when Integer/MAX_VALUE is given as a radius).
199                for (Direction d : Direction.DIAGONALS) {
200                    shadowCast(1, 1.0, 0.0, 0, d.deltaX, d.deltaY, 0, rad, startX, startY, decay, light, resistanceMap, radiusTechnique);
201                    shadowCast(1, 1.0, 0.0, d.deltaX, 0, 0, d.deltaY, rad, startX, startY, decay, light, resistanceMap, radiusTechnique);
202                }
203                break;
204        }
205
206        return light;
207    }

209        /**
210     * Calculates the Field Of View for the provided map from the given x, y
211     * coordinates. Returns a light map where the values represent a percentage
212     * of fully lit.
213     *
214     * The starting point for the calculation is considered to be at the center
215     * of the origin cell. Radius determinations are determined by the provided
216     * RadiusStrategy. A conical section of FOV is lit by this method if
217     * span is greater than 0.
218     *
219     * @param resistanceMap the grid of cells to calculate on; the kind made by DungeonUtility.generateResistances()
220     * @param startX the horizontal component of the starting location
221     * @param startY the vertical component of the starting location
222     * @param radius the distance the light will extend to
223     * @param radiusTechnique provides a means to calculate the radius as desired
224     * @param angle the angle in degrees that will be the center of the FOV cone, 0 points right
225     * @param span the angle in degrees that measures the full arc contained in the FOV cone
226     * @return the computed light grid
227     */
228    public double[][] calculateFOV(double[][] resistanceMap, int startX, int startY, double radius,
229                                   Radius radiusTechnique, double angle, double span) {
230
231        double rad = Math.max(1, radius);
232
233        double decay = 1.0 / rad;
234
235                double angle2 = Math.toRadians((angle > 360.0 || angle < 0.0)
236                                ? GwtCompatibility.IEEEremainder(angle + 720.0, 360.0) : angle);
237        double span2 = Math.toRadians(span);
238        int width = resistanceMap.length;
239        int height = resistanceMap[0].length;
240
241        initializeLightMap(width, height);
242        light[startX][startY] = 1;//make the starting space full power
243
244        switch (type) {
245            case RIPPLE:
246            case RIPPLE_LOOSE:
247            case RIPPLE_TIGHT:
248            case RIPPLE_VERY_LOOSE:
249                initializeNearLight(width, height);
250                doRippleFOV(light, rippleValue(type), startX, startY, startX, startY, decay, rad, resistanceMap, nearLight, radiusTechnique, angle2, span2);
251                break;
252            case SHADOW:
253                // this should be fixed now, sorta. the distance is checked in the method this calls, so it doesn't ever
254                // run through more than (width + height) iterations of the radius-related loop (which seemed to be the
255                // only problem, running through billions of iterations when Integer/MAX_VALUE is given as a radius).
256                int ctr = 0;
257                boolean started = false;
258                for (Direction d : ccw) {
259                    ctr %= 4;
260                    ++ctr;
261                    if (angle <= Math.PI / 2.0 * ctr + span / 2.0)
262                        started = true;
263                    if (started) {
264                        if(ctr < 4 && angle < Math.PI / 2.0 * (ctr - 1) - span / 2.0)
265                            break;
266                        light = shadowCastLimited(1, 1.0, 0.0, 0, d.deltaX, d.deltaY, 0, rad, startX, startY, decay, light, resistanceMap, radiusTechnique, angle2, span2);
267                        light = shadowCastLimited(1, 1.0, 0.0, d.deltaX, 0, 0, d.deltaY, rad, startX, startY, decay, light, resistanceMap, radiusTechnique, angle2, span2);
268                    }
269                }
270                break;
271        }
272
273        return light;
274    }

342
343        double rad = Math.max(1, radius);
344        double decay = 1.0 / rad;
345
346        light[startX][startY] = 1;//make the starting space full power
347
348
349        shadowCast(1, 1.0, 0.0, 0, 1, 1, 0, rad, startX, startY, decay, light, resistanceMap, radiusTechnique);

352        shadowCast(1, 1.0, 0.0, 0, 1, -1, 0, rad, startX, startY, decay, light, resistanceMap, radiusTechnique);
353        shadowCast(1, 1.0, 0.0, 1, 0, 0, -1, rad, startX, startY, decay, light, resistanceMap, radiusTechnique);

355        shadowCast(1, 1.0, 0.0, 0, -1, -1, 0, rad, startX, startY, decay, light, resistanceMap, radiusTechnique);
356        shadowCast(1, 1.0, 0.0, -1, 0, 0, -1, rad, startX, startY, decay, light, resistanceMap, radiusTechnique);
357
358        shadowCast(1, 1.0, 0.0, 0, -1, 1, 0, rad, startX, startY, decay, light, resistanceMap, radiusTechnique);
359        shadowCast(1, 1.0, 0.0, -1, 0, 0, 1, rad, startX, startY, decay, light, resistanceMap, radiusTechnique);
360        return light;

367         * @param height
368         *            The height that {@link #light} should have.

370        private void initializeLightMap(int width, int height) {
371                if (light == null)
372                        light = new double[width][height];
373                else {
374                        if (light.length != width || light[0].length != height)
375                                /* Size changed */
376                                light = new double[width][height];
377                        else {
378                                /*
379                                 * Size did not change, we simply need to erase the previous
380                                 * result
381                                 */
382                                ArrayTools.fill(light, 0.0);
383                        }
384                }
385        }

411                switch (type) {

426    private static void doRippleFOV(double[][] lightMap, int ripple, int x, int y, int startx, int starty, double decay, double radius, double[][] map, boolean[][] indirect, Radius radiusStrategy) {
427        /* Not using Deque's interface, it isn't GWT compatible */
428        final LinkedList<Coord> dq = new LinkedList<>();
429        int width = lightMap.length;
430        int height = lightMap[0].length;
431        dq.offer(Coord.get(x, y));
432        while (!dq.isEmpty()) {
433            Coord p = dq.removeFirst();
434            if (lightMap[p.x][p.y] <= 0 || indirect[p.x][p.y]) {
435                continue;//no light to spread
436            }
437
438            for (Direction dir : Direction.OUTWARDS) {
439                int x2 = p.x + dir.deltaX;
440                int y2 = p.y + dir.deltaY;
441                if (x2 < 0 || x2 >= width || y2 < 0 || y2 >= height //out of bounds
442                        || radiusStrategy.radius(startx, starty, x2, y2) >= radius + 1) {//+1 to cover starting tile
443                    continue;
444                }
445
446                double surroundingLight = nearRippleLight(x2, y2, ripple, startx, starty, decay, lightMap, map, indirect, radiusStrategy);
447                if (lightMap[x2][y2] < surroundingLight) {
448                    lightMap[x2][y2] = surroundingLight;
449                    if (map[x2][y2] < 1) {//make sure it's not a wall
450                        dq.offer(Coord.get(x2, y2));//redo neighbors since this one's light changed
451                    }
452                }
453            }
454        }
455    }

459    private static void doRippleFOV(double[][] lightMap, int ripple, int x, int y, int startx, int starty, double decay, double radius, double[][] map, boolean[][] indirect, Radius radiusStrategy, double angle, double span) {
460        /* Not using Deque's interface, it isn't GWT compatible */
461        final LinkedList<Coord> dq = new LinkedList<>();
462        int width = lightMap.length;
463        int height = lightMap[0].length;
464        dq.offer(Coord.get(x, y));
465        while (!dq.isEmpty()) {
466            Coord p = dq.removeFirst();
467            if (lightMap[p.x][p.y] <= 0 || indirect[p.x][p.y]) {
468                continue;//no light to spread
469            }
470
471            for (Direction dir : ccw_full) {
472                int x2 = p.x + dir.deltaX;
473                int y2 = p.y + dir.deltaY;
474                if (x2 < 0 || x2 >= width || y2 < 0 || y2 >= height //out of bounds
475                        || radiusStrategy.radius(startx, starty, x2, y2) >= radius + 1) {//+1 to cover starting tile
476                    continue;
477                }
478                double newAngle = Math.atan2(y2 - starty, x2 - startx) + Math.PI * 2;
479                                if (Math.abs(GwtCompatibility.IEEEremainder(angle - newAngle + Math.PI * 8, Math.PI * 2)) > span / 2.0)
480                                        continue;
481
482                double surroundingLight = nearRippleLight(x2, y2, ripple, startx, starty, decay, lightMap, map, indirect, radiusStrategy );
483                if (lightMap[x2][y2] < surroundingLight) {
484                    lightMap[x2][y2] = surroundingLight;
485                    if (map[x2][y2] < 1) {//make sure it's not a wall
486                        dq.offer(Coord.get(x2, y2));//redo neighbors since this one's light changed
487                    }
488                }
489            }
490        }

493    private static double nearRippleLight(int x, int y, int rippleNeighbors, int startx, int starty, double decay, double[][] lightMap, double[][] map, boolean[][] indirect, Radius radiusStrategy) {
494        if (x == startx && y == starty) {
495            return 1;
496        }
497        int width = lightMap.length;
498        int height = lightMap[0].length;
499        List<Coord> neighbors = new ArrayList<>();
500        double tmpDistance = 0, testDistance;
501        Coord c;
502        for (Direction di : Direction.OUTWARDS) {
503            int x2 = x + di.deltaX;
504            int y2 = y + di.deltaY;
505            if (x2 >= 0 && x2 < width && y2 >= 0 && y2 < height) {
506                tmpDistance = radiusStrategy.radius(startx, starty, x2, y2);
507                int idx = 0;
508                for(int i = 0; i < neighbors.size() && i <= rippleNeighbors; i++)
509                {
510                    c = neighbors.get(i);
511                    testDistance = radiusStrategy.radius(startx, starty, c.x, c.y);
512                    if(tmpDistance < testDistance) {
513                        break;
514                    }
515                    idx++;
516                }
517                neighbors.add(idx, Coord.get(x2, y2));
518            }
519        }
520
521        if (neighbors.isEmpty()) {
522            return 0;
523        }
524        neighbors = neighbors.subList(0, Math.min(neighbors.size(), rippleNeighbors));
525/*
526        while (neighbors.size() > rippleNeighbors) {
527            Coord p = neighbors.remove(0);
528            double dist = radiusStrategy.radius(startx, starty, p.x, p.y);
529            double dist2 = 0;
530            for (Coord p2 : neighbors) {
531                dist2 = Math.max(dist2, radiusStrategy.radius(startx, starty, p2.x, p2.y));
532            }
533            if (dist < dist2) {//not the largest, put it back
534                neighbors.add(p);
535            }
536        }
537*/
538        double light = 0;
539        int lit = 0, indirects = 0;
540        for (Coord p : neighbors) {
541            if (lightMap[p.x][p.y] > 0) {
542                lit++;
543                if (indirect[p.x][p.y]) {
544                    indirects++;
545                }
546                double dist = radiusStrategy.radius(x, y, p.x, p.y);
547                light = Math.max(light, lightMap[p.x][p.y] - dist * decay - map[p.x][p.y]);
548            }
549        }
550
551        if (map[x][y] >= 1 || indirects >= lit) {
552            indirect[x][y] = true;
553        }
554        return light;
555    }

557    private static double[][] shadowCast(int row, double start, double end, int xx, int xy, int yx, int yy,
558                                  double radius, int startx, int starty, double decay, double[][] lightMap,
559                                  double[][] map, Radius radiusStrategy) {
560        double newStart = 0;
561        if (start < end) {
562            return lightMap;
563        }
564        int width = lightMap.length;
565        int height = lightMap[0].length;
566
567        boolean blocked = false;
568        for (int distance = row; distance <= radius && distance < width + height && !blocked; distance++) {
569            int deltaY = -distance;
570            for (int deltaX = -distance; deltaX <= 0; deltaX++) {
571                int currentX = startx + deltaX * xx + deltaY * xy;
572                int currentY = starty + deltaX * yx + deltaY * yy;
573                double leftSlope = (deltaX - 0.5f) / (deltaY + 0.5f);
574                double rightSlope = (deltaX + 0.5f) / (deltaY - 0.5f);
575
576                if (!(currentX >= 0 && currentY >= 0 && currentX < width && currentY < height) || start < rightSlope) {
577                    continue;
578                } else if (end > leftSlope) {
579                    break;
580                }
581                double deltaRadius = radiusStrategy.radius(deltaX, deltaY);
582                //check if it's within the lightable area and light if needed
583                if (deltaRadius <= radius) {
584                    double bright = 1 - decay * deltaRadius;
585                    lightMap[currentX][currentY] = bright;
586                }
587
588                if (blocked) { //previous cell was a blocking one
589                    if (map[currentX][currentY] >= 1) {//hit a wall
590                        newStart = rightSlope;
591                    } else {
592                        blocked = false;
593                        start = newStart;
594                    }
595                } else {
596                    if (map[currentX][currentY] >= 1 && distance < radius) {//hit a wall within sight line
597                        blocked = true;
598                        lightMap = shadowCast(distance + 1, start, leftSlope, xx, xy, yx, yy, radius, startx, starty, decay, lightMap, map, radiusStrategy);
599                        newStart = rightSlope;
600                    }
601                }
602            }
603        }
604        return lightMap;
605    }
606    private static double[][] shadowCastLimited(int row, double start, double end, int xx, int xy, int yx, int yy,
607                                         double radius, int startx, int starty, double decay, double[][] lightMap,
608                                         double[][] map, Radius radiusStrategy, double angle, double span) {
609        double newStart = 0;
610        if (start < end) {
611            return lightMap;
612        }
613        int width = lightMap.length;
614        int height = lightMap[0].length;
615
616        boolean blocked = false;
617        for (int distance = row; distance <= radius && distance < width + height && !blocked; distance++) {
618            int deltaY = -distance;
619            for (int deltaX = -distance; deltaX <= 0; deltaX++) {
620                int currentX = startx + deltaX * xx + deltaY * xy;
621                int currentY = starty + deltaX * yx + deltaY * yy;
622                double leftSlope = (deltaX - 0.5f) / (deltaY + 0.5f);
623                double rightSlope = (deltaX + 0.5f) / (deltaY - 0.5f);
624
625                if (!(currentX >= 0 && currentY >= 0 && currentX < width && currentY < height) || start < rightSlope) {
626                    continue;
627                } else if (end > leftSlope) {
628                    break;
629                }
630                double newAngle = Math.atan2(currentY - starty, currentX - startx) + Math.PI * 2;
631                                if (Math.abs(GwtCompatibility.IEEEremainder(angle - newAngle + Math.PI * 8, Math.PI * 2)) > span / 2.0)
632                                        continue;
633
634                double deltaRadius = radiusStrategy.radius(deltaX, deltaY);
635                //check if it's within the lightable area and light if needed
636                if (deltaRadius <= radius) {
637                    double bright = 1 - decay * deltaRadius;
638                    lightMap[currentX][currentY] = bright;
639                }
640
641                if (blocked) { //previous cell was a blocking one
642                    if (map[currentX][currentY] >= 1) {//hit a wall
643                        newStart = rightSlope;
644                    } else {
645                        blocked = false;
646                        start = newStart;
647                    }
648                } else {
649                    if (map[currentX][currentY] >= 1 && distance < radius) {//hit a wall within sight line
650                        blocked = true;
651                        lightMap = shadowCastLimited(distance + 1, start, leftSlope, xx, xy, yx, yy, radius, startx, starty, decay, lightMap, map, radiusStrategy, angle, span);
652                        newStart = rightSlope;
653                    }
654                }
655            }
656        }
657        return lightMap;
658    }

660    /**
661     * Adds multiple FOV maps together in the simplest way possible; does not check line-of-sight between FOV maps.
662     * Clamps the highest value for any single position at 1.0.
663     * @param maps an array or vararg of 2D double arrays, each usually returned by calculateFOV()
664     * @return the sum of all the 2D double arrays passed, using the dimensions of the first if they don't all match
665     */
666    public static double[][] addFOVs(double[][]... maps)
667    {
668        if(maps == null || maps.length == 0)
669            return new double[0][0];
670        double[][] map = ArrayTools.copy(maps[0]);
671        for(int i = 1; i < maps.length; i++)
672        {
673            for (int x = 0; x < map.length && x < maps[i].length; x++) {
674                for (int y = 0; y < map[x].length && y < maps[i][x].length; y++) {
675                    map[x][y] += maps[i][x][y];
676                }
677            }
678        }
679        for (int x = 0; x < map.length; x++) {
680            for (int y = 0; y < map[x].length; y++) {
681                if(map[x][y] > 1.0) map[x][y] = 1.0;
682            }
683        }
684        return map;
685    }

688     * Adds multiple FOV maps together in the simplest way possible; does not check line-of-sight between FOV maps.
689     * Clamps the highest value for any single position at 1.0.
690     * @param maps an Iterable of 2D double arrays (most collections implement Iterable),
691     *             each usually returned by calculateFOV()
692     * @return the sum of all the 2D double arrays passed, using the dimensions of the first if they don't all match
693     */
694    public static double[][] addFOVs(Iterable<double[][]> maps)
695    {
696        if(maps == null)
697            return new double[0][0];
698        Iterator<double[][]> it = maps.iterator();
699        if(!it.hasNext())
700            return new double[0][0];
701        double[][] map = ArrayTools.copy(it.next()), t;
702        while (it.hasNext())
703        {
704            t = it.next();
705            for (int x = 0; x < map.length && x < t.length; x++) {
706                for (int y = 0; y < map[x].length && y < t[x].length; y++) {
707                    map[x][y] += t[x][y];
708                }
709            }
710        }
711        for (int x = 0; x < map.length; x++) {
712            for (int y = 0; y < map[x].length; y++) {
713                if(map[x][y] > 1.0) map[x][y] = 1.0;
714            }
715        }
716        return map;
717    }

719    /**
720     * Adds together multiple FOV maps, but only adds to a position if it is visible in the given LOS map. Useful if
721     * you want distant lighting to be visible only if the player has line-of-sight to a lit cell. Typically the LOS map
722     * is calculated by calculateLOSMap(), using the same resistance map used to calculate the FOV maps.
723     * Clamps the highest value for any single position at 1.0.
724     * @param losMap an LOS map such as one generated by calculateLOSMap()
725     * @param maps an array or vararg of 2D double arrays, each usually returned by calculateFOV()
726     * @return the sum of all the 2D double arrays in maps where a cell was visible in losMap
727     */
728    public static double[][] mixVisibleFOVs(double[][] losMap, double[][]... maps)
729    {
730        if(losMap == null || losMap.length == 0)
731            return addFOVs(maps);
732        double[][] map = new double[losMap.length][losMap[0].length];
733        if(maps == null || maps.length == 0)
734            return map;
735        for(int i = 0; i < maps.length; i++)
736        {
737            for (int x = 0; x < losMap.length && x < map.length && x < maps[i].length; x++) {
738                for (int y = 0; y < losMap[x].length && y < map[x].length && y < maps[i][x].length; y++) {
739                    if(losMap[x][y] > 0.0001) {
740                        map[x][y] += maps[i][x][y];
741                        if(map[x][y] > 1.0) map[x][y] = 1.0;
742                    }
743                }
744            }
745        }
746        return map;
747    }

786     * mixVisibleFOVs() to limit extra light sources to those visible from the starting point. Just like calling
787     * calculateFOV(), this creates a new double[][]; there doesn't appear to be a way to work with Ripple FOV and avoid

795    {

797            return new double[0][0];
798        return calculateFOV(resistanceMap, startX, startY, resistanceMap.length + resistanceMap[0].length, Radius.SQUARE);

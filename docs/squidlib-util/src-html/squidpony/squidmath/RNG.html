010/**
011 * A wrapper class for working with random number generators in a more friendly way.
012 * Implements {@link IRNG}, which covers most of the API surface, but RNG implements
013 * a decent amount of additional methods. You should consider if your code needs
014 * these additional methods, and if not you should use IRNG as the type for when you
015 * need some random number generator.
016 * <p>
017 * Includes methods for getting values between two numbers and for getting
018 * random elements from a collection or array. There are methods to shuffle
019 * a collection and to get a random ordering that can be applied as one shuffle
020 * across multiple collections, such as via {@link OrderedMap#reorder(int...)},
021 * {@link ArrayTools#reorder(ArrayList, int...)}, and so on. You can construct
022 * an RNG with all sorts of RandomnessSource implementations, and choosing them
023 * is usually not a big concern because the default works very well. If you target
024 * GWT, then it is suggested that you use {@link GWTRNG} instead of RNG; both
025 * implement {@link IRNG}, which is enough for most usage across SquidLib, but
026 * GWTRNG is optimized heavily for better performance on GWT, even returning long
027 * values faster than implementations that natively do their math on longs. It has
028 * worse performance on 64-bit PCs and mobile devices, but should also have better
029 * performance on 32-bit PCs and mobile devices.
030 * <br>
031 * But if you do want advice on what RandomnessSource to use... {@link LinnormRNG}
032 * is the default, and is the fastest generator that passes stringent tests, but
033 * relative to many of the others it has a significantly shorter period (the
034 * amount of random  numbers it will go through before repeating the sequence),
035 * at {@code pow(2, 64)} as opposed to {@link XoRoRNG}'s {@code pow(2, 128) - 1},
036 * or {@link LongPeriodRNG}'s {@code pow(2, 1024) - 1}. {@link LightRNG} is a solid
037 * choice and the former default RandomnessSource; additional features of LightRNG
038 * are exposed in {@link MoonwalkRNG} and using MoonwalkRNG is recommended if you
039 * need unusual features like skipping backwards in a random number sequence, taking
040 * a result of a nextLong() call and reversing it to get the state that produced it,
041 * or calculating the distance in number of nextLong() calls between two results of
042 * nextLong() calls. LightRNG is a StatefulRandomness, which lets it be used in
043 * {@link StatefulRNG}, and so is LinnormRNG, but LightRNG is also a
044 * {@link SkippingRandomness}, which means you can leap forward or backwards in
045 * its sequence very efficiently (LinnormRNG is not a SkippingRandomness).
046 * {@link ThrustAltRNG} provides similar qualities to LightRNG,
047 * and can be faster, but can't produce all possible 64-bit values (possibly some 32-bit
048 * values as well); it was the default at one point so you may want to keep compatibility
049 * with some versions by specifying ThrustAltRNG (before and after ThrustAltRNG was the
050 * default, LightRNG was used). For most cases, you should decide between
051 * LinnormRNG, ThrustAltRNG, LightRNG, LongPeriodRNG, and XoRoRNG based on your priorities.
052 * LinnormRNG is the best if you want high speed, very good quality of randomness, and
053 * expect to either generate less than 18446744073709551616 numbers or don't care if
054 * patterns appear after you generate that many numbers. LightRNG is the second-best at
055 * the above criteria, but is the best option if you need an RNG that can skip backwards
056 * or jump forwards without incurring speed penalties. LongPeriodRNG is best if you for some
057 * reason need a massive amount of random numbers (as in, ten quintillion would be far too
058 * little) or want to split up such a large generator into unrelated subsequences. XoRoRNG
059 * is best if you want good speed and quality but need to generate more than
060 * 18446744073709551616 numbers, though less than 340282366920938463463374607431768211456
061 * numbers. ThrustAltRNG is faster than LinnormRNG usually, but its inability to generate
062 * some outputs may make it a poor choice for some usage (it also has some bias toward
063 * specific numbers and produces them more frequently, but not frequently enough to make it
064 * fail statistical tests). {@link JabRNG} and {@link LapRNG} are competitors in the
065 * no-quality-guarantees department; JabRNG is slightly slower than LapRNG but faster than
066 * ThrustAltRNG, actually can pass a fair amount of quality tests, and can't produce a
067 * majority of all possible 64-bit values. Its period is half that of ThrustAltRNG and
068 * LightRNG, at 2 to the 63. LapRNG doesn't pass any tests, really, but it can fool a human
069 * observer; its period is probably 2 to the 65 and it's incredibly fast but offers no
070 * special features.
071 * <br>
072 * There are many more RandomnessSource implementations! You might want significantly less
073 * predictable random results, which  {@link IsaacRNG} can provide, along with a
074 * large period. The quality of {@link PermutedRNG} is also good, usually, and it
075 * has a sound basis in PCG-Random, an involved library with many variants on its
076 * RNGs.
077 * There may be reasons why you would want a random number generator that uses 32-bit
078 * math instead of the more common 64-bit math, but using a 32-bit int on desktop and
079 * Android won't act the same as that same 32-bit int on GWT. Since GWT is stuck with
080 * JavaScript's implementation of ints with doubles, overflow (which is needed for an
081 * RNG) doesn't work with ints as expected, but does with GWT's implementation of longs.
082 * If targeting GWT, {@link Lathe32RNG} is significantly faster at producing int values
083 * than any long-based generator, and will produce the same results on GWT as on desktop
084 * or Android (not all 32-bit generators do this). {@link ThrustAlt32RNG},
085 * {@link Zag32RNG}, and {@link Oriole32RNG} are also GWT-safe, but other generators that
086 * were thought to be GWT-friendly are not. These GWT-unsafe generators have other uses,
087 * but should not be used on GWT: {@link PintRNG}, {@link Light32RNG},
088 * and {@link FlapRNG}. All other generators use longs, and so will be
089 * slower than the recommended Lathe32RNG on GWT, but much faster on 64-bit JREs.
090 * @author Eben Howard - http://squidpony.com - howard@squidpony.com
091 * @author Tommy Ettinger
092 * @author smelC
093 */

176        /**
177         * Generates the next pseudorandom number. Subclasses should
178         * override this, as this is used by all other methods.
179         * <p>
180         * <p>The general contract of {@code next} is that it returns an
181         * {@code int} value and if the argument {@code bits} is between
182         * {@code 1} and {@code 32} (inclusive), then that many low-order
183         * bits of the returned value will be (approximately) independently
184         * chosen bit values, each of which is (approximately) equally
185         * likely to be {@code 0} or {@code 1}. The method {@code next} is
186         * implemented by class {@code Random} by atomically updating the seed to
187         * <pre>{@code (seed * 0x5DEECE66DL + 0xBL) & ((1L << 48) - 1)}</pre>
188         * and returning
189         * <pre>{@code (int)(seed >>> (48 - bits))}.</pre>
190         *
191         * This is a linear congruential pseudorandom number generator, as
192         * defined by D. H. Lehmer and described by Donald E. Knuth in
193         * <i>The Art of Computer Programming,</i> Volume 3:
194         * <i>Seminumerical Algorithms</i>, section 3.2.1.
195         *
196         * @param bits random bits
197         * @return the next pseudorandom value from this random number
198         * generator's sequence
199         * @since 1.1
200         */
201        @Override
202        protected int next(int bits) {
203            return randomnessSource.next(bits);
204        }

263    /**
264     * Returns the average of a number of randomly selected numbers from the
265     * provided range, with min being inclusive and max being exclusive. It will
266     * sample the number of times passed in as the third parameter.
267     * <p>
268     * The inclusive and exclusive behavior is to match the behavior of the
269     * similar method that deals with floating point values.
270     * <p>
271     * This can be used to weight RNG calls to the average between min and max.
272     *
273     * @param min     the minimum bound on the return value (inclusive)
274     * @param max     the maximum bound on the return value (exclusive)
275     * @param samples the number of samples to take
276     * @return the found value
277     */
278    public int betweenWeighted(int min, int max, int samples) {
279        int sum = 0;
280        for (int i = 0; i < samples; i++) {
281            sum += between(min, max);
282        }
283
284        return Math.round((float) sum / samples);
285    }

320     * Returns a random element from the provided ShortSet. If the set is empty
321     * then an exception is thrown.
322     * <p>
323     * <p>
324     * Requires iterating through a random amount of the elements in set, so performance depends on the size of set but
325     * is likely to be decent. This is mostly meant for internal use, the same as ShortSet.
326     * </p>
327     *
328     * @param set the ShortSet to get an element from
329     * @return the randomly selected element
330     */
331    public short getRandomElement(ShortSet set) {
332        if (set.size <= 0) {
333            throw new UnsupportedOperationException("ShortSet cannot be empty when getting a random element");
334        }
335        int n = nextInt(set.size);
336        short s = 0;
337        ShortSet.ShortSetIterator ssi = set.iterator();
338        while (n-- >= 0 && ssi.hasNext)
339            s = ssi.next();
340        ssi.reset();
341        return s;
342    }

398    /**
399     * Given a {@link List} l, this selects a random element of l to be the first value in the returned list l2. It
400     * retains the order of elements in l after that random element and makes them follow the first element in l2, and
401     * loops around to use elements from the start of l after it has placed the last element of l into l2.
402     * <br>
403     * Essentially, it does what it says on the tin. It randomly rotates the List l.
404     * <br>
405     * If you only need to iterate through a collection starting at a random point, the method getRandomStartIterable()
406     * should have better performance. This was GWT incompatible before GWT 2.8.0 became the version SquidLib uses; now
407     * this method works fine with GWT.
408     *
409     * @param l   A {@link List} that will not be modified by this method. All elements of this parameter will be
410     *            shared with the returned List.
411     * @param <T> No restrictions on type. Changes to elements of the returned List will be reflected in the parameter.
412     * @return A shallow copy of {@code l} that has been rotated so its first element has been randomly chosen
413     * from all possible elements but order is retained. Will "loop around" to contain element 0 of l after the last
414     * element of l, then element 1, etc.
415     */
416    public <T> List<T> randomRotation(final List<T> l) {
417        final int sz = l.size();
418        if (sz == 0)
419            return Collections.<T>emptyList();
420
421                /*
422                 * Collections.rotate should prefer the best-performing way to rotate l,
423                 * which would be an in-place modification for ArrayLists and an append
424                 * to a sublist for Lists that don't support efficient random access.
425                 */
426        List<T> l2 = new ArrayList<>(l);
427        Collections.rotate(l2, nextInt(sz));
428        return l2;
429    }

431    /**
432     * Get an Iterable that starts at a random location in list and continues on through list in its current order.
433     * Loops around to the beginning after it gets to the end, stops when it returns to the starting location.
434     * <br>
435     * You should not modify {@code list} while you use the returned reference. And there'll be no
436     * ConcurrentModificationException to detect such erroneous uses.
437     *
438     * @param list A list <b>with a constant-time {@link List#get(int)} method</b> (otherwise performance degrades).
439     * @return An {@link Iterable} that iterates over {@code list} but start at
440     * a random index. If the chosen index is {@code i}, the iterator
441     * will return:
442     * {@code list[i]; list[i+1]; ...; list[list.length() - 1]; list[0]; list[i-1]}
443     */
444    public <T> Iterable<T> getRandomStartIterable(final List<T> list) {
445        final int sz = list.size();
446        if (sz == 0)
447            return Collections.<T>emptyList();
448
449                /*
450                 * Here's a tricky bit: Defining 'start' here means that every Iterator
451                 * returned by the returned Iterable will have the same iteration order.
452                 * In other words, if you use more than once the returned Iterable,
453                 * you'll will see elements in the same order every time, which is
454                 * desirable.
455                 */
456        final int start = nextInt(sz);
457
458        return new Iterable<T>() {
459            @Override
460            public Iterator<T> iterator() {
461                return new Iterator<T>() {
462
463                    int next = -1;
464
465                    @Override
466                    public boolean hasNext() {
467                        return next != start;
468                    }
469
470                    @Override
471                    public T next() {
472                        if (next == start)
473                            throw new NoSuchElementException("Iteration terminated; check hasNext() before next()");
474                        if (next == -1)
475                                        /* First call */
476                            next = start;
477                        final T result = list.get(next);
478                        if (next == sz - 1)
479                                        /*
480                                         * Reached the list's end, let's continue from the list's
481                                         * left.
482                                         */
483                            next = 0;
484                        else
485                            next++;
486                        return result;
487                    }
488
489                    @Override
490                    public void remove() {
491                        throw new UnsupportedOperationException("Remove is not supported from a randomStartIterable");
492                    }
493
494                    @Override
495                    public String toString() {
496                        return "RandomStartIterator at index " + next;
497                    }
498                };
499            }
500        };
501    }

516    /**
517     * Shuffle an array using the Fisher-Yates algorithm and returns a shuffled copy.
518     * GWT-compatible since GWT 2.8.0, which is the default if you use libGDX 1.9.5 or higher.
519     * <br>
520     * <a href="https://en.wikipedia.org/wiki/Fisher%E2%80%93Yates_shuffle">Wikipedia has more on this algorithm</a>.
521     *
522     * @param elements an array of T; will not be modified
523     * @param <T>      can be any non-primitive type.
524     * @return a shuffled copy of elements
525     */
526    @Override
527    public <T> T[] shuffle(final T[] elements) {
528        final int size = elements.length;
529        final T[] array = Arrays.copyOf(elements, size);
530        for (int i = size; i > 1; i--) {
531            swap(array, i - 1, nextIntHasty(i));
532        }
533        return array;
534    }

555    /**
556     * Shuffle an array using the Fisher-Yates algorithm. If possible, create a new array or reuse an existing array
557     * with the same length as elements and pass it in as dest; the dest array will contain the shuffled contents of
558     * elements and will also be returned as-is.
559     * <br>
560     * <a href="https://en.wikipedia.org/wiki/Fisher%E2%80%93Yates_shuffle">Wikipedia has more on this algorithm</a>.
561     *
562     * @param elements an array of T; will not be modified
563     * @param <T>      can be any non-primitive type.
564     * @param dest     Where to put the shuffle. If it does not have the same length as {@code elements}, this will use the
565     *                 randomPortion method of this class to fill the smaller dest.
566     * @return {@code dest} after modifications
567     */
568    @Override
569    public <T> T[] shuffle(T[] elements, T[] dest) {
570        if (dest.length != elements.length)
571            return randomPortion(elements, dest);
572        System.arraycopy(elements, 0, dest, 0, elements.length);
573        shuffleInPlace(dest);
574        return dest;
575    }

591     * Shuffles a {@link Collection} of T using the Fisher-Yates algorithm. The result
592     * is allocated if {@code buf} is null or if {@code buf} isn't empty,
593     * otherwise {@code elements} is poured into {@code buf}.
594     * <br>
595     * <a href="https://en.wikipedia.org/wiki/Fisher%E2%80%93Yates_shuffle">Wikipedia has more on this algorithm</a>.
596     * @param elements a Collection of T; will not be modified
597     * @param <T>      can be any non-primitive type.
598     * @param buf a buffer as an ArrayList that will be filled with the shuffled contents of elements;
599     *            if null or non-empty, a new ArrayList will be allocated and returned
600     * @return a shuffled ArrayList containing the whole of elements in pseudo-random order.
601     */
602    @Override
603    public <T> ArrayList<T> shuffle(Collection<T> elements, /*@Nullable*/ ArrayList<T> buf) {
604        final ArrayList<T> al;
605        if (buf == null || !buf.isEmpty())
606            al = new ArrayList<>(elements);
607        else {
608            al = buf;
609            al.addAll(elements);
610        }
611        final int n = al.size();
612        for (int i = n; i > 1; i--) {
613            Collections.swap(al, nextInt(i), i - 1);
614        }
615        return al;
616    }

693     * May return an empty array if the parameters are invalid (end is less than/equal to start, or start is negative).
694     *
695     * @param start the start of the range of numbers to potentially use (inclusive)
696     * @param end   the end of the range of numbers to potentially use (exclusive)
697     * @param count the total number of elements to use; will be less if the range is smaller than count
698     * @return an int array that contains at most one of each number in the range
699     */
700    public int[] randomRange(int start, int end, int count) {
701        if (end <= start || start < 0)
702            return new int[0];
703
704        int n = end - start;
705        final int[] data = new int[n];
706
707        for (int e = start, i = 0; e < end; e++) {
708            data[i++] = e;
709        }
710
711        for (int i = 0; i < n - 1; i++) {
712            final int r = i + nextInt(n - i), t = data[r];
713            data[r] = data[i];
714            data[i] = t;
715        }
716        final int[] array = new int[Math.min(count, n)];
717        System.arraycopy(data, 0, array, 0, Math.min(count, n));
718        return array;

739     * Gets a pseudo-random double from the Gaussian distribution, which will usually be between -1.0 and 1.0 but is not
740     * always in that range. If you do want values to always be between -1 and 1 and a float is OK, consider using
741     * {@link #nextCurvedFloat()}, which is a different distribution that is less sharply-curved towards 0 and
742     * terminates at -1 and 1.
743     * @return a value from the Gaussian distribution
744     */
745    public double nextGaussian() {
746        if (haveNextNextGaussian) {
747            haveNextNextGaussian = false;
748            return nextNextGaussian;
749        } else {
750            double v1, v2, s;
751            do {
752                v1 = 2 * nextDouble() - 1; // between -1 and 1
753                v2 = 2 * nextDouble() - 1; // between -1 and 1
754                s = v1 * v1 + v2 * v2;
755            } while (s >= 1 || s == 0);
756            double multiplier = Math.sqrt(-2 * Math.log(s) / s);
757            nextNextGaussian = v2 * multiplier;
758            haveNextNextGaussian = true;
759            return v1 * multiplier;
760        }
761    }

838     * Exclusive on bound (which must be positive), with an inner bound of 0.
839     * If bound is negative or 0 this always returns 0.
840     * <br>
841     * Credit for this method goes to <a href="https://oroboro.com/large-random-in-range/">Rafael Baptista's blog</a>,
842     * with some adaptation for signed long values and a 64-bit generator. This method is drastically faster than the
843     * previous implementation when the bound varies often (roughly 4x faster, possibly more). It also always gets
844     * exactly one random number, so it advances the state as much as {@link #nextInt(int)}.
845     * @param bound the outer exclusive bound; should be positive, otherwise this always returns 0L
846     * @return a random long between 0 (inclusive) and bound (exclusive)
847     */
848    public long nextLong(long bound) {
849        long rand = random.nextLong();
850        if (bound <= 0) return 0;
851        final long randLow = rand & 0xFFFFFFFFL;
852        final long boundLow = bound & 0xFFFFFFFFL;
853        rand >>>= 32;
854        bound >>= 32;
855        final long z = (randLow * boundLow >> 32);
856        final long t = rand * boundLow + z;
857        return rand * bound + (t >> 32) + ((t & 0xFFFFFFFFL) + randLow * bound >> 32) - (z >> 63);
858    }

1234    /**
1235     * Gets a random portion of data (an array), assigns that portion to output (an array) so that it fills as much as
1236     * it can, and then returns output. Will only use a given position in the given data at most once; does this by
1237     * generating random indices for data's elements, but only as much as needed, assigning the copied section to output
1238     * and not modifying data.
1239     * <br>
1240     * Based on http://stackoverflow.com/a/21460179 , credit to Vincent van der Weele; modifications were made to avoid
1241     * copying or creating a new generic array (a problem on GWT).
1242     *
1243     * @param data   an array of T; will not be modified.
1244     * @param output an array of T that will be overwritten; should always be instantiated with the portion length
1245     * @param <T>    can be any non-primitive type.
1246     * @return an array of T that has length equal to output's length and may contain unchanged elements (null if output
1247     * was empty) if data is shorter than output
1248     */
1249    public <T> T[] randomPortion(T[] data, T[] output) {
1250        int length = data.length;
1251        int n = Math.min(length, output.length);
1252        int[] mapping = ArrayTools.range(n);
1253        for (int i = 0; i < n; i++) {
1254            int r = nextIntHasty(length);
1255            output[i] = data[mapping[r]];
1256            mapping[r] = mapping[--length];
1257        }
1258
1259        return output;
1260    }

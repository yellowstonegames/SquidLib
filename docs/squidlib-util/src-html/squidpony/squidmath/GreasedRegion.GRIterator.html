013/**
014 * Region encoding of on/off information about areas using bitsets; uncompressed (fatty), but fast (greased lightning).
015 * This can handle any size of 2D data, and is not strictly limited to 256x256 as CoordPacker is. It stores several long
016 * arrays and uses each bit in one of those numbers to represent a single point, though sometimes this does waste bits
017 * if the height of the area this encodes is not a multiple of 64 (if you store a 80x64 map, this uses 80 longs; if you
018 * store an 80x65 map, this uses 160 longs, 80 for the first 64 rows and 80 more to store the next row). It's much
019 * faster than CoordPacker at certain operations (anything that expands or retracts an area, including
020 * {@link #expand()}), {@link #retract()}), {@link #fringe()}), {@link #surface()}, and {@link #flood(GreasedRegion)},
021 * and slightly faster on others, like {@link #and(GreasedRegion)} (called intersectPacked() in CoordPacker) and
022 * {@link #or(GreasedRegion)} (called unionPacked() in CoordPacker).
023 * <br>
024 * Each GreasedRegion is mutable, and instance methods typically modify that instance and return it for chaining. There
025 * are exceptions, usually where multiple GreasedRegion values are returned and the instance is not modified.
026 * <br>
027 * Typical usage involves constructing a GreasedRegion from some input data, like a char[][] for a map or a double[][]
028 * from DijkstraMap, and modifying it spatially with expand(), retract(), flood(), etc. It's common to mix in data from
029 * other GreasedRegions with and() (which gets the intersection of two GreasedRegions and stores it in one), or() (which
030 * is like and() but for the union), xor() (like and() but for exclusive or, finding only cells that are on in exactly
031 * one of the two GreasedRegions), and andNot() (which can be considered the "subtract another region from me" method).
032 * There are 8-way (Chebyshev distance) variants on all of the spatial methods, and methods without "8way" in the name
033 * are either 4-way (Manhattan distance) or not affected by distance measurement. Once you have a GreasedRegion, you may
034 * want to get a single random point from it (use {@link #singleRandom(RNG)}), get several random points from it (use
035 * {@link #randomPortion(RNG, int)} for random sampling or {@link #randomSeparated(double, RNG)} for points that have
036 * some distance between each other), or get all points from it (use {@link #asCoords()}. You may also want to produce
037 * some 2D data from one or more GreasedRegions, such as with {@link #sum(GreasedRegion...)} or {@link #toChars()}. The
038 * most effective techniques regarding GreasedRegion involve multiple methods, like getting a few random points from an
039 * existing GreasedRegion representing floor tiles in a dungeon with {@link #randomPortion(RNG, int)}, then inserting
040 * those into a new GreasedRegion with {@link #insertSeveral(Coord...)}, and then finding a random expansion of those
041 * initial points with {@link #spill(GreasedRegion, int, RNG)}, giving the original GreasedRegion of floor tiles as the
042 * first argument. This could be used to position puddles of water or patches of mold in a dungeon level, while still
043 * keeping the starting points and finished points within the boundaries of valid (floor) cells.
044 * <br>
045 * For efficiency, you can place one GreasedRegion into another (typically a temporary value that is no longer needed
046 * and can be recycled) using {@link #remake(GreasedRegion)}, or give the information that would normally be used to
047 * construct a fresh GreasedRegion to an existing one of the same dimensions with {@link #refill(boolean[][])} or any
048 * of the overloads of refill(). These re-methods don't do as much work as a constructor does if the width and height
049 * of their argument are identical to their current width and height, and don't create more garbage for the GC.
050 * <br>
051 * Created by Tommy Ettinger on 6/24/2016.
052 */

077    /**
078     * Constructs a GreasedRegion with the given rectangular boolean array, with width of bits.length and height of
079     * bits[0].length, any value of true considered "on", and any value of false considered "off."
080     * @param bits a rectangular 2D boolean array where true is on and false is off
081     */
082    public GreasedRegion(final boolean[][] bits)
083    {
084        width = bits.length;
085        height = bits[0].length;
086        ySections = (height + 63) >> 6;
087        yEndMask = -1L >>> (64 - (height & 63));
088        data = new long[width * ySections];
089        for (int x = 0; x < width; x++) {
090            for (int y = 0; y < height; y++) {
091                if(bits[x][y]) data[x * ySections + (y >> 6)] |= 1L << (y & 63);
092            }
093        }
094    }
095    /**
096     * Reassigns this GreasedRegion with the given rectangular boolean array, reusing the current data storage (without
097     * extra allocations) if this.width == map.length and this.height == map[0].length. The current values stored in
098     * this are always cleared, then any value of true in map is considered "on", and any value of false in map is
099     * considered "off."
100     * @param map a rectangular 2D boolean array where true is on and false is off
101     * @return this for chaining
102     */
103    public GreasedRegion refill(final boolean[][] map) {
104        if (map != null && map.length > 0 && width == map.length && height == map[0].length) {
105            Arrays.fill(data, 0L);
106            for (int x = 0; x < width; x++) {
107                for (int y = 0; y < height; y++) {
108                    data[x * ySections + (y >> 6)] |= (map[x][y] ? 1L : 0L) << (y & 63);
109                }
110            }
111            return this;
112        } else {
113            width = (map == null) ? 0 : map.length;
114            height = (map == null || map.length <= 0) ? 0 : map[0].length;
115            ySections = (height + 63) >> 6;
116            yEndMask = -1L >>> (64 - (height & 63));
117            data = new long[width * ySections];
118            for (int x = 0; x < width; x++) {
119                for (int y = 0; y < height; y++) {
120                    if(map[x][y]) data[x * ySections + (y >> 6)] |= 1L << (y & 63);
121                }
122            }
123            return this;
124        }
125    }

031    /**
032     * Default constructor; uses SplitMix64, which is of high quality, but low period (which rarely matters for games),
033     * and has good speed, tiny state size, and excellent 64-bit number generation.
034     * <br>
035     * Compatibility note: previous versions of SquidLib used Mersenne Twister by default. Due to the incompatibility
036     * of the threads used by this Mersenne Twister implementation with GWT and HTML5 applications, the randomness
037     * algorithm has been changed to a faster, more compatible algorithm, though it does suffer from a much lower
038     * period. If you need drastically larger periods than 2^64, you can pass a LongPeriodRNG (or MersenneTwister on

040     * period of a PRNG is, you probably don't need to worry about it; it's mainly relevant to heavily multi-threaded
041     * applications anyway. The addition of LongPeriodRNG on March 21, 2016 should help to take the part of a fast,
042     * large-period RNG, which MersenneTwister is unable to act as on GWT. The default may change again some time after
043     * May 1, 2016, now that we have XoRoRNG, which is approximately as fast as LightRNG and has a substantially better
044     * period (pow(2, 128) - 1).
045     */

104        /**
105         * Generates the next pseudorandom number. Subclasses should
106         * override this, as this is used by all other methods.
107         * <p>
108         * <p>The general contract of {@code next} is that it returns an
109         * {@code int} value and if the argument {@code bits} is between
110         * {@code 1} and {@code 32} (inclusive), then that many low-order
111         * bits of the returned value will be (approximately) independently
112         * chosen bit values, each of which is (approximately) equally
113         * likely to be {@code 0} or {@code 1}. The method {@code next} is
114         * implemented by class {@code Random} by atomically updating the seed to
115         * <pre>{@code (seed * 0x5DEECE66DL + 0xBL) & ((1L << 48) - 1)}</pre>
116         * and returning
117         * <pre>{@code (int)(seed >>> (48 - bits))}.</pre>
118         * <p>
119         * This is a linear congruential pseudorandom number generator, as
120         * defined by D. H. Lehmer and described by Donald E. Knuth in
121         * <i>The Art of Computer Programming,</i> Volume 3:
122         * <i>Seminumerical Algorithms</i>, section 3.2.1.
123         *
124         * @param bits random bits
125         * @return the next pseudorandom value from this random number
126         * generator's sequence
127         * @since 1.1
128         */
129        @Override
130        protected int next(int bits) {
131            return randomnessSource.next(bits);
132        }
133    }

184    /**
185     * Returns the average of a number of randomly selected numbers from the
186     * provided range, with min being inclusive and max being exclusive. It will
187     * sample the number of times passed in as the third parameter.
188     *
189     * The inclusive and exclusive behavior is to match the behavior of the
190     * similar method that deals with floating point values.
191     *
192     * This can be used to weight RNG calls to the average between min and max.
193     *
194     * @param min the minimum bound on the return value (inclusive)
195     * @param max the maximum bound on the return value (exclusive)
196     * @param samples the number of samples to take
197     * @return the found value
198     */
199    public int betweenWeighted(int min, int max, int samples) {
200        int sum = 0;
201        for (int i = 0; i < samples; i++) {
202            sum += between(min, max);
203        }
204
205        return Math.round((float) sum / samples);
206    }

238    /**
239     * Returns a random element from the provided ShortSet. If the set is empty
240     * then an exception is thrown.
241     *
242     * <p>
243     * Requires iterating through a random amount of the elements in set, so performance depends on the size of set but
244     * is likely to be decent. This is mostly meant for internal use, the same as ShortSet.
245     * </p>
246     * @param set the ShortSet to get an element from
247     * @return the randomly selected element
248     */
249    public short getRandomElement(ShortSet set) {
250        if (set.size <= 0) {
251            throw new UnsupportedOperationException("ShortSet cannot be empty when getting a random element");
252        }
253        int n = nextInt(set.size);
254        short s = 0;
255        ShortSet.ShortSetIterator ssi = set.iterator();
256        while (n-- >= 0 && ssi.hasNext)
257            s = ssi.next();
258        ssi.reset();
259        return s;
260    }

263     * Returns a random element from the provided Collection, which should have predictable iteration order if you want
264     * predictable behavior for identical RNG seeds, though it will get a random element just fine for any Collection
265     * (just not predictably in all cases). If you give this a Set, it should be a LinkedHashSet or some form of sorted
266     * Set like TreeSet if you want predictable results. Any List or Queue should be fine. Map does not implement
267     * Collection, thank you very much Java library designers, so you can't actually pass a Map to this, though you can
268     * pass the keys or values. If coll is empty, returns null.
269     *
270     * <p>
271     * Requires iterating through a random amount of coll's elements, so performance depends on the size of coll but is
272     * likely to be decent, as long as iteration isn't unusually slow. This replaces {@code getRandomElement(Queue)},
273     * since Queue implements Collection and the older Queue-using implementation was probably less efficient.
274     * </p>
275     * @param <T> the type of the returned object
276     * @param coll the Collection to get an element from; remember, Map does not implement Collection
277     * @return the randomly selected element
278     */
279    public <T> T getRandomElement(Collection<T> coll) {
280        if (coll.size() <= 0) {
281            return null;
282        }
283        int n = nextInt(coll.size());
284        T t = null;
285        Iterator<T> it = coll.iterator();
286        while (n-- >= 0 && it.hasNext())
287            t = it.next();
288        return t;
289    }

312        /**
313     * Given a {@link List} l, this selects a random element of l to be the first value in the returned list l2. It
314     * retains the order of elements in l after that random element and makes them follow the first element in l2, and
315     * loops around to use elements from the start of l after it has placed the last element of l into l2.
316     * <br>
317     * Essentially, it does what it says on the tin. It randomly rotates the List l.
318     * <br>
319     * If you only need to iterate through a collection starting at a random point, the method getRandomStartIterable()
320     * should have better performance.
321     *
322         * @param l A {@link List} that will not be modified by this method. All elements of this parameter will be
323     *            shared with the returned List.
324     * @param <T> No restrictions on type. Changes to elements of the returned List will be reflected in the parameter.
325     * @return A shallow copy of {@code l} that has been rotated so its first element has been randomly chosen
326     * from all possible elements but order is retained. Will "loop around" to contain element 0 of l after the last
327     * element of l, then element 1, etc.
328         */
329    @GwtIncompatible /* Because of Collections.rotate */
330        public <T> List<T> randomRotation(final List<T> l) {
331                final int sz = l.size();
332                if (sz == 0)
333                        return Collections.<T>emptyList();
334
335                /*
336                 * Collections.rotate should prefer the best-performing way to rotate l, which would be an in-place
337                 * modification for ArrayLists and an append to a sublist for Lists that don't support efficient random access.
338                 */
339        List<T> l2 = new ArrayList<>(l);
340        Collections.rotate(l2, nextInt(sz));
341        return l2;
342        }
343    /**
344     * Get an Iterable that starts at a random location in list and continues on through list in its current order.
345     * Loops around to the beginning after it gets to the end, stops when it returns to the starting location.
346     * <br>
347     * You should not modify {@code list} while you use the returned reference. And there'll be no
348     * ConcurrentModificationException to detect such erroneous uses.
349     * @param list A list <b>with a constant-time {@link List#get(int)} method</b> (otherwise performance degrades).
350     * @return An {@link Iterable} that iterates over {@code list} but start at
351     *         a random index. If the chosen index is {@code i}, the iterator
352     *         will return:
353     *         {@code list[i]; list[i+1]; ...; list[list.length() - 1]; list[0]; list[i-1]}
354     *
355     */
356    public <T> Iterable<T> getRandomStartIterable(final List<T> list) {
357        final int sz = list.size();
358        if (sz == 0)
359            return Collections.<T> emptyList();
360
361                /*
362                 * Here's a tricky bit: Defining 'start' here means that every Iterator
363                 * returned by the returned Iterable will have the same iteration order.
364                 * In other words, if you use more than once the returned Iterable,
365                 * you'll will see elements in the same order every time, which is
366                 * desirable.
367                 */
368        final int start = nextInt(sz);
369
370        return new Iterable<T>() {
371            @Override
372            public Iterator<T> iterator() {
373                return new Iterator<T>() {
374
375                    int next = -1;
376
377                    @Override
378                    public boolean hasNext() {
379                        return next != start;
380                    }
381
382                    @Override
383                    public T next() {
384                        if (next == start)
385                            throw new NoSuchElementException("Iteration terminated; check hasNext() before next()");
386                        if (next == -1)
387                                        /* First call */
388                            next = start;
389                        final T result = list.get(next);
390                        if (next == sz - 1)
391                                        /*
392                                         * Reached the list's end, let's continue from the list's
393                                         * left.
394                                         */
395                            next = 0;
396                        else
397                            next++;
398                        return result;
399                    }
400
401                    @Override
402                                        public void remove() {
403                        throw new UnsupportedOperationException("Remove is not supported from a randomStartIterable");
404                                        }
405
406                                        @Override
407                    public String toString() {
408                        return "RandomStartIterator at index " + next;
409                    }
410                };
411            }
412        };
413    }

415    /**
416     * Shuffle an array using the Fisher-Yates algorithm. Not GWT-compatible; use the overload that takes two arrays.
417     * <br>
418     * https://en.wikipedia.org/wiki/Fisher%E2%80%93Yates_shuffle
419     * @param elements an array of T; will not be modified
420     * @param <T> can be any non-primitive type.
421     * @return a shuffled copy of elements
422     */
423    @GwtIncompatible
424    public <T> T[] shuffle(T[] elements)
425    {
426        int n = elements.length;
427        T[] array = Arrays.copyOf(elements, n);
428        for (int i = 0; i < n; i++)
429        {
430            int r = i + nextInt(n - i);
431            T t = array[r];
432            array[r] = array[i];
433            array[i] = t;
434        }
435        return array;
436    }

438    /**
439     * Shuffle an array using the "inside-out" Fisher-Yates algorithm. DO NOT give the same array for both elements and
440     * dest, since the prior contents of dest are rearranged before elements is used, and if they refer to the same
441     * array, then you can end up with bizarre bugs where one previously-unique item shows up dozens of times. If
442     * possible, create a new array with the same length as elements and pass it in as dest; the returned value can be
443     * assigned to whatever you want and will have the same items as the newly-formed array.
444     * <br>
445     * https://en.wikipedia.org/wiki/Fisher%E2%80%93Yates_shuffle#The_.22inside-out.22_algorithm
446     * @param elements an array of T; will not be modified
447     * @param <T> can be any non-primitive type.
448     * @param dest Where to put the shuffle. If it does not have the same length as {@code elements}, this will use the
449     *             randomPortion method of this class to fill the smaller dest. MUST NOT be the same array as elements!
450     * @return {@code dest} after modifications
451     */
452    /* This method has this prototype to be compatible with GWT. */
453    public <T> T[] shuffle(T[] elements, T[] dest)
454    {
455        if (dest.length != elements.length)
456            return randomPortion(elements, dest);
457        for (int i = 0; i < elements.length; i++)
458        {
459            int r = nextInt(i + 1);
460            if(r != i)
461                dest[i] = dest[r];
462            dest[r] = elements[i];
463        }
464        return dest;
465    }

467        /**
468     * Shuffles a {@link Collection} of T using the Fisher-Yates algorithm and returns an ArrayList of T.
469     * @param elements a Collection of T; will not be modified
470     * @param <T> can be any non-primitive type.
471     * @return a shuffled ArrayList containing the whole of elements in pseudo-random order.
472     */
473    public <T> ArrayList<T> shuffle(Collection<T> elements)
474    {
475        ArrayList<T> al = new ArrayList<>(elements);
476        int n = al.size();
477        for (int i = 0; i < n; i++)
478        {
479            Collections.swap(al, i + nextInt(n - i), i);
480        }
481        return al;
482    }

580    /**
581     * Gets a random subrange of the non-negative ints from start (inclusive) to end (exclusive), using count elements.
582     * May return an empty array if the parameters are invalid (end is less than/equal to start, or start is negative).
583     * @param start the start of the range of numbers to potentially use (inclusive)
584     * @param end  the end of the range of numbers to potentially use (exclusive)
585     * @param count the total number of elements to use; will be less if the range is smaller than count
586     * @return an int array that contains at most one of each number in the range
587     */
588    public int[] randomRange(int start, int end, int count)
589    {
590        if(end <= start || start < 0)
591            return new int[0];
592
593        int n = end - start;
594        int[] data = new int[n];
595
596        for (int e = start, i = 0; e < end; e++) {
597            data[i++] = e;
598        }
599
600        for (int i = 0; i < n; i++)
601        {
602            int r = i + nextInt(n - i);
603            int t = data[r];
604            data[r] = data[i];
605            data[i] = t;
606        }
607        int[] array = new int[Math.min(count, n)];
608        System.arraycopy(data, 0, array, 0, Math.min(count, n));
609        return array;
610    }

612    /**
613     * @return a value from the gaussian distribution
614     */
615    public synchronized double nextGaussian() {
616        if (haveNextNextGaussian) {
617            haveNextNextGaussian = false;
618            return nextNextGaussian;
619        } else {
620            double v1, v2, s;
621            do {
622                v1 = 2 * nextDouble() - 1; // between -1 and 1
623                v2 = 2 * nextDouble() - 1; // between -1 and 1
624                s = v1 * v1 + v2 * v2;
625            } while (s >= 1 || s == 0);
626            double multiplier = Math.sqrt(-2 * Math.log(s) / s);
627            nextNextGaussian = v2 * multiplier;
628            haveNextNextGaussian = true;
629            return v1 * multiplier;
630        }
631    }

633    /**
634     * This returns a maximum of 0.9999999999999999 because that is the largest
635     * Double value that is less than 1.0
636     *
637     * @return a value between 0 (inclusive) and 0.9999999999999999 (inclusive)
638     */
639    public double nextDouble() {
640        return (random.nextLong() & 0x1fffffffffffffL) * DOUBLE_UNIT;
641        // consider changing to this in a future version; it will break compatibility but should be fast/correct
642        //return Double.longBitsToDouble(0x3FFL << 52 | random.nextLong() >>> 12) - 1.0;
643
644    }

681    /**
682     * Returns a random long below the given bound, or 0 if the bound is 0 or
683     * negative.
684     *
685     * @param bound the upper bound (exclusive)
686     * @return the found number
687     */
688    public long nextLong( final long bound ) {
689        if ( bound <= 0 ) return 0;
690        long threshold = (0x7fffffffffffffffL - bound + 1) % bound;
691        for (;;) {
692            long bits = random.nextLong() & 0x7fffffffffffffffL;
693            if (bits >= threshold)
694                return bits % bound;
695        }
696    }
697    /**
698     * Returns a random non-negative integer below the given bound, or 0 if the bound is 0 or
699     * negative.
700     *
701     * @param bound the upper bound (exclusive)
702     * @return the found number
703     */
704    public int nextInt(final int bound) {
705        if ( bound <= 0 ) return 0;
706        int threshold = (0x7fffffff - bound + 1) % bound;
707        for (;;) {
708            int bits = random.next(31);
709            if (bits >= threshold)
710                return bits % bound;
711        }
712    }
713    /**
714     * Returns a random non-negative integer below the given bound, or 0 if the bound is 0.
715     * Uses an aggressively optimized technique that has some bias, but mostly for values of
716     * bound over 1 billion. This method is considered "hasty" since it should be faster than
717     * nextInt() but gives up some statistical quality to do so. It also has undefined behavior
718     * if bound is negative, though it will probably produce a negative number (just how
719     * negative is an open question).
720     * <br>
721     * Credit goes to Daniel Lemire, http://lemire.me/blog/2016/06/27/a-fast-alternative-to-the-modulo-reduction/
722     * @param bound the upper bound (exclusive); behavior is undefined if bound is negative
723     * @return the found number
724     */
725    public int nextIntHasty(final int bound) {
726        return (int)((bound * (random.nextLong() & 0x7FFFFFFFL)) >> 31);
727    }

729    /**
730     * Gets a random Coord that has x between 0 (inclusive) and width (exclusive) and y between 0 (inclusive)
731     * and height (exclusive). This makes one call to randomLong to generate (more than) 31 random bits for
732     * each axis, and should be very fast. Remember that Coord values are cached in a pool that starts able to
733     * hold up to 255 x and 255 y for positive values, and the pool should be grown with the static method
734     * Coord.expandPool() in order to efficiently use larger Coord values. If width and height are very large,
735     * greater than 100,000 for either, this particular method may show bias toward certain positions due to
736     * the "hasty" technique used to reduce the random numbers to the given size, but because most maps in
737     * tile-based games are relatively small, this technique should be fine.
738     * <br>
739     * Credit goes to Daniel Lemire, http://lemire.me/blog/2016/06/27/a-fast-alternative-to-the-modulo-reduction/
740     * @param width the upper bound (exclusive) for x coordinates
741     * @param height the upper bound (exclusive) for y coordinates
742     * @return a random Coord between (0,0) inclusive and (width,height) exclusive
743     */
744    public Coord nextCoord(int width, int height)
745    {
746        final long n = random.nextLong();
747        return Coord.get((int)((width * (n >>> 33)) >> 31), (int)((height * (n & 0x7FFFFFFFL)) >> 31));
748    }

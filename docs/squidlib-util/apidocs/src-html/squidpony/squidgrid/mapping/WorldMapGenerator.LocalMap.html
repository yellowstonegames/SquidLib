014/**
015 * Can be used to generate world maps with a wide variety of data, starting with height, temperature and moisture.
016 * From there, you can determine biome information in as much detail as your game needs, with default implementations
017 * available; one assigns a single biome to each cell based on heat/moisture, and the other gives a gradient between two
018 * biome types for every cell. The maps this produces with {@link SphereMap} are valid for spherical world projections,
019 * while the maps from {@link TilingMap} are for toroidal world projections and will wrap from edge to opposite edge
020 * seamlessly thanks to <a href="https://www.gamedev.net/blog/33/entry-2138456-seamless-noise/">a technique from the
021 * Accidental Noise Library</a> that involves getting a 2D slice of 4D Simplex noise. Because of how Simplex noise
022 * works, this also allows extremely high zoom levels for all types of map as long as certain parameters are within
023 * reason. Other world maps produce more conventional shapes, like {@link SpaceViewMap} and {@link RotatingSpaceMap}
024 * make a view of a marble-like world from space, and others make more unconventional shapes, like {@link EllipticalMap}
025 * or {@link EllipticalHammerMap}, which form a 2:1 ellipse shape that accurately keeps sizes but not relative shapes,
026 * {@link RoundSideMap}, which forms a pill-shape, and {@link HyperellipticalMap}, which takes parameters so it can fit
027 * any shape between a circle or ellipse and a rectangle (the default is a slightly squared-off ellipse). You can access
028 * the height map with the {@link #heightData} field, the heat map with the {@link #heatData} field, the moisture map
029 * with the {@link #moistureData} field, and a special map that stores ints representing the codes for various ranges of
030 * elevation (0 to 8 inclusive, with 0 the deepest ocean and 8 the highest mountains) with {@link #heightCodeData}. The
031 * last map should be noted as being the simplest way to find what is land and what is water; any height code 4 or
032 * greater is land, and any height code 3 or less is water.
033 * <br>
034 * Biome mapping is likely to need customization per-game, but some good starting points are {@link SimpleBiomeMapper},
035 * which stores one biome per cell, and {@link DetailedBiomeMapper}, which gives each cell a midway value between two
036 * biomes.
037 */

231     * Generates a world using the specified RNG state as a long, with specific land and heat modifiers that affect
232     * the land-water ratio and the average temperature, respectively.
233     * The worlds this produces will always have width and height as specified in the constructor (default 256x256).
234     * You can call {@link #zoomIn(int, int, int)} to double the resolution and center on the specified area, but the width
235     * and height of the 2D arrays this changed, such as {@link #heightData} and {@link #moistureData} will be the same.
236     * @param landMod 1.0 is Earth-like, less than 1 is more-water, more than 1 is more-land; a random value will be used if this is negative
237     * @param heatMod 1.125 is Earth-like, less than 1 is cooler, more than 1 is hotter; a random value will be used if this is negative
238     * @param state the state to give this generator's RNG; if the same as the last call, this will reuse data
239     */
240    public void generate(double landMod, double heatMod, long state)
241    {
242        if(cacheA != (int) (state & 0xFFFFFFFFL) || cacheB != (int) (state >>> 32) ||
243                landMod != landModifier || heatMod != heatModifier)
244        {
245            seedA = (int) (state & 0xFFFFFFFFL);
246            seedB = (int) (state >>> 32);
247            zoom = 0;
248            startCacheX.clear();
249            startCacheY.clear();
250            startCacheX.add(0);
251            startCacheY.add(0);
252            zoomStartX = width >> 1;
253            zoomStartY = height >> 1;
254
255        }
256        //System.out.printf("generate, zoomStartX: %d, zoomStartY: %d\n", zoomStartX, zoomStartY);
257
258        regenerate(startX = (zoomStartX >> zoom) - (width >> 1 + zoom), startY = (zoomStartY >> zoom) - (height >> 1 + zoom),
259                //startCacheX.peek(), startCacheY.peek(),
260                usedWidth = (width >> zoom), usedHeight = (height >> zoom), landMod, heatMod, seedA, seedB);
261    }

273    /**
274     * Halves the resolution of the map and doubles the area it covers repeatedly, halving {@code zoomAmount} times; the
275     * 2D arrays this uses keep their sizes. This version of zoomOut allows you to specify where the zoom should be
276     * centered, using the current coordinates (if the map size is 256x256, then coordinates should be between 0 and
277     * 255, and will refer to the currently used area and not necessarily the full world size).
278     * <br>
279     * Only has an effect if you have previously zoomed in using {@link #zoomIn(int, int, int)} or its overload.
280     * @param zoomCenterX the center X position to zoom out from; if too close to an edge, this will stop moving before it would extend past an edge
281     * @param zoomCenterY the center Y position to zoom out from; if too close to an edge, this will stop moving before it would extend past an edge
282     */
283    public void zoomOut(int zoomAmount, int zoomCenterX, int zoomCenterY)
284    {
285        zoomAmount = Math.min(zoom, zoomAmount);
286        if(zoomAmount == 0) return;
287        if(zoomAmount < 0) {
288            zoomIn(-zoomAmount, zoomCenterX, zoomCenterY);
289            return;
290        }
291        if(zoom > 0)
292        {
293            if(cacheA != seedA || cacheB != seedB)
294            {
295                generate(rng.nextLong());
296            }
297            zoomStartX = Math.min(Math.max(
298                    (zoomStartX + (zoomCenterX - (width >> 1))) >> zoomAmount,
299                    width >> 1), (width << zoom - zoomAmount) - (width >> 1));
300            zoomStartY = Math.min(Math.max(
301                    (zoomStartY + (zoomCenterY - (height >> 1))) >> zoomAmount,
302                    height >> 1), (height << zoom - zoomAmount) - (height >> 1));
303//            System.out.printf("zoomOut, zoomStartX: %d, zoomStartY: %d\n", zoomStartX, zoomStartY);
304            zoom -= zoomAmount;
305            startCacheX.pop();
306            startCacheY.pop();
307            startCacheX.add(Math.min(Math.max(startCacheX.pop() + (zoomCenterX >> zoom + 1) - (width >> zoom + 2),
308                    0), width - (width >> zoom)));
309            startCacheY.add(Math.min(Math.max(startCacheY.pop() + (zoomCenterY >> zoom + 1) - (height >> zoom + 2),
310                    0), height - (height >> zoom)));
311//            zoomStartX = Math.min(Math.max((zoomStartX >> 1) + (zoomCenterX >> zoom + 1) - (width >> zoom + 2),
312//                    0), width - (width >> zoom));
313//            zoomStartY = Math.min(Math.max((zoomStartY >> 1) + (zoomCenterY >> zoom + 1) - (height >> zoom + 2),
314//                    0), height - (height >> zoom));
315            regenerate(startX = (zoomStartX >> zoom) - (width >> zoom + 1), startY = (zoomStartY >> zoom) - (height >> zoom + 1),
316                    //startCacheX.peek(), startCacheY.peek(),
317                    usedWidth = width >> zoom,  usedHeight = height >> zoom,
318                    landModifier, heatModifier, cacheA, cacheB);
319            rng.setState(cacheA, cacheB);
320        }
321
322    }

335    /**
336     * Doubles the resolution of the map and halves the area it covers repeatedly, doubling {@code zoomAmount} times;
337     * the 2D arrays this uses keep their sizes. This version of zoomIn allows you to specify where the zoom should be
338     * centered, using the current coordinates (if the map size is 256x256, then coordinates should be between 0 and
339     * 255, and will refer to the currently used area and not necessarily the full world size).
340     * <br>
341     * Although there is no technical restriction on maximum zoom, zooming in more than 5 times (64x scale or greater)
342     * will make the map appear somewhat less realistic due to rounded shapes appearing more bubble-like and less like a
343     * normal landscape.
344     * @param zoomCenterX the center X position to zoom in to; if too close to an edge, this will stop moving before it would extend past an edge
345     * @param zoomCenterY the center Y position to zoom in to; if too close to an edge, this will stop moving before it would extend past an edge
346     */
347    public void zoomIn(int zoomAmount, int zoomCenterX, int zoomCenterY)
348    {
349        if(zoomAmount == 0) return;
350        if(zoomAmount < 0)
351        {
352            zoomOut(-zoomAmount, zoomCenterX, zoomCenterY);
353            return;
354        }
355        if(seedA != cacheA || seedB != cacheB)
356        {
357            generate(rng.nextLong());
358        }
359        zoomStartX = Math.min(Math.max(
360                (zoomStartX + zoomCenterX - (width >> 1) << zoomAmount),
361                width >> 1), (width << zoom + zoomAmount) - (width >> 1));
362//        int oldZoomY = zoomStartY;
363        zoomStartY = Math.min(Math.max(
364                (zoomStartY + zoomCenterY - (height >> 1) << zoomAmount),
365                height >> 1), (height << zoom + zoomAmount) - (height >> 1));
366//        System.out.printf("zoomIn, zoomStartX: %d, zoomStartY: %d, oldZoomY: %d, unedited: %d, upperCap: %d\n", zoomStartX, zoomStartY,
367//                oldZoomY, (oldZoomY + zoomCenterY - (height >> 1) << zoomAmount), (height << zoom + zoomAmount) - (height >> 1));
368        zoom += zoomAmount;
369        if(startCacheX.isEmpty())
370        {
371            startCacheX.add(0);
372            startCacheY.add(0);
373        }
374        else {
375            startCacheX.add(Math.min(Math.max(startCacheX.peek() + (zoomCenterX >> zoom - 1) - (width >> zoom + 1),
376                    0), width - (width >> zoom)));
377            startCacheY.add(Math.min(Math.max(startCacheY.peek() + (zoomCenterY >> zoom - 1) - (height >> zoom + 1),
378                    0), height - (height >> zoom)));
379        }
380        regenerate(startX = (zoomStartX >> zoom) - (width >> 1 + zoom), startY = (zoomStartY >> zoom) - (height >> 1 + zoom),
381                //startCacheX.peek(), startCacheY.peek(),
382                usedWidth = width >> zoom, usedHeight = height >> zoom,
383                landModifier, heatModifier, cacheA, cacheB);
384        rng.setState(cacheA, cacheB);
385    }

407    {
408        if(high < deepWaterUpper)
409            return 0;
410        if(high < mediumWaterUpper)
411            return 1;
412        if(high < shallowWaterUpper)
413            return 2;
414        if(high < coastalWaterUpper)
415            return 3;
416        if(high < sandUpper)
417            return 4;
418        if(high < grassUpper)
419            return 5;
420        if(high < forestUpper)
421            return 6;
422        if(high < rockUpper)
423            return 7;
424        return 8;
425    }

008/**
009 * This input processing class can handle mouse and keyboard input, using a squidpony.squidgrid.gui.gdx.SquidMouse for
010 * Mouse input and a user implementation of the SquidInput.KeyHandler interface to react to keys represented as chars
011 * and the modifiers those keys were pressed with, any of alt, ctrl, and/or shift. Not all keys are representable by
012 * default in unicode, so symbolic representations are stored in constants in this class, and are passed to
013 * {@link KeyHandler#handle(char, boolean, boolean, boolean)} as chars like DOWN_ARROW or its value, '\u2193'. Shift
014 * modifies the input as it would on a QWERTY keyboard, and the exact mapping is documented in
015 * {@link #fromCode(int, boolean)} as well. This class handles mouse input immediately, but stores keypresses in a
016 * queue, storing all key events and allowing them to be processed one at a time using {@link #next()} or all at once
017 * using {@link #drain()}. To have an effect, it needs to be registered by calling
018 * {@link Input#setInputProcessor(InputProcessor)}. Note that calling {@link #hasNext()} does more than just check if
019 * there are events that can be processed; because hasNext() is expected to be called frequently, it is also the point
020 * where this class checks if a key is being held and so the next event should occur. Holding a key only causes the
021 * keyDown() method of InputListener to be called once, so this uses hasNext() to see if there should be a next event
022 * coming from a held key.
023 * <br>
024 * This also allows some key remapping, including remapping so a key pressed with modifiers like Ctrl and Shift could
025 * act like '?' (which could be used by expert players to avoid accidentally opening a help menu they don't need), and
026 * that would free up '?' for some other use that could also be remapped. The remap() methods do much of this, often
027 * with help from {@link #combineModifiers(char, boolean, boolean, boolean)}, while the unmap() methods allow removal of
028 * any no-longer-wanted remappings.
029 * <br>
030 * It does not perform the blocking functionality of earlier SquidKey implementations, because this is meant to run
031 * in an event-driven libGDX game and should not step on the toes of libGDX's input handling. To block game logic
032 * until an event has been received, check hasNext() in the game's render() method and effectively "block" by not
033 * running game logic if hasNext() returns false. You can process an event if hasNext() returns true by calling 
034 * {@link #next()}. Mouse inputs do not affect hasNext(), and next() will process only key pressed events. Also, see
035 * above about the extra behavior of hasNext regarding held keys.
036 *
037 * @author Eben Howard - http://squidpony.com - howard@squidpony.com
038 * @author Nathan Sweet
039 * @author Tommy Ettinger
040 * */
041public class SquidInput extends InputAdapter {
042    /**
043     * A single-method interface used to process "typed" characters, special characters produced by unusual keys, and
044     * modifiers that can affect them. SquidInput has numerous static char values that are expected to be passed
045     * to handle() in place of the special keys (such as arrow keys) that do not have a standard char value.
046     */
047    public interface KeyHandler{
048        /**
049         * The only method you need to implement yourself in KeyHandler, this should react to keys such as
050         * 'a' (produced by pressing the A key while not holding Shift), 'E' (produced by pressing the E key while
051         * holding Shift), and '\u2190' (left arrow in unicode, also available as a constant in SquidInput, produced by
052         * pressing the left arrow key even though that key does not have a default unicode representation). Capital
053         * letters will be capitalized when they are passed to this, but they may or may not have the shift argument as
054         * true depending on how this method was called. Symbols that may be produced by holding Shift and pressing a
055         * number or a symbol key can vary between keyboards (some may require Shift to be held down, others may not).
056         * <br>
057         * This can react to the input in whatever way you find appropriate for your game.
058         * @param key a char of the "typed" representation of the key, such as 'a' or 'E', or if there is no Unicode
059         *            character for the key, an appropriate alternate character as documented in SquidInput.fromKey()
060         * @param alt true if the Alt modifier was being held while this key was entered, false otherwise.
061         * @param ctrl true if the Ctrl modifier was being held while this key was entered, false otherwise.
062         * @param shift true if the Shift modifier was being held while this key was entered, false otherwise.
063         */
064        void handle(char key, boolean alt, boolean ctrl, boolean shift);
065    }

270    /**
271     * Remaps a char that could be input and processed by {@link KeyHandler#handle(char, boolean, boolean, boolean)}
272     * (possibly with some pressed modifiers) to another char with possible modifiers. When the first key/modifier mix
273     * is received, it will be translated to the second group in this method.
274     * @param pressedChar a source char that might be used in handling, like 'q', 'A', '7', '(', or {@link #UP_ARROW}.
275     * @param pressedAlt true if alt is part of the source combined keypress, false otherwise
276     * @param pressedCtrl true if ctrl is part of the source combined keypress, false otherwise
277     * @param pressedShift true if shift is part of the source combined keypress, false otherwise
278     * @param targetChar a target char that might be used in handling, like 'q', 'A', '7', '(', or {@link #UP_ARROW}.
279     * @param targetAlt true if alt is part of the target combined keypress, false otherwise
280     * @param targetCtrl true if ctrl is part of the target combined keypress, false otherwise
281     * @param targetShift true if shift is part of the target combined keypress, false otherwise
282     * @return this for chaining
283     */
284    public SquidInput remap(char pressedChar, boolean pressedAlt, boolean pressedCtrl, boolean pressedShift,
285                            char targetChar, boolean targetAlt, boolean targetCtrl, boolean targetShift)
286    {
287        mapping.put(combineModifiers(pressedChar, pressedAlt, pressedCtrl, pressedShift),
288                combineModifiers(targetChar, targetAlt, targetCtrl, targetShift));
289        return this;
290    }

305    /**
306     * Remaps many keypress combinations, each of which is a char and several potential modifiers, to other keypress
307     * combinations. When the first of a pair of combinations is received, it will be translated to the second
308     * combination of the pair.
309     * @see #combineModifiers(char, boolean, boolean, boolean) combineModifiers is usually used to make the contents of pairs
310     * @param pairs an int array alternating source and target keypresses, each probably produced by {@link #combineModifiers(char, boolean, boolean, boolean)}
311     * @return this for chaining
312     */
313    public SquidInput remap(int[] pairs)
314    {
315        int len;
316        if(pairs == null || (len = pairs.length) <= 1)
317            return this;
318        for (int i = 0; i < len - 1; i++) {
319            mapping.put(pairs[i], pairs[++i]);
320        }
321        return this;
322    }

359    /**
360     * Combines the key (as it would be given to {@link KeyHandler#handle(char, boolean, boolean, boolean)}) with the
361     * three booleans for the alt, ctrl, and shift modifier keys, returning an int that can be used with the internal
362     * queue of ints or the public {@link #mapping} of received inputs to actual inputs the program can process.
363     * @param key a char that might be used in handling, like 'q', 'A', '7', '(', or {@link #UP_ARROW}.
364     * @param alt true if alt is part of this combined keypress, false otherwise
365     * @param ctrl true if ctrl is part of this combined keypress, false otherwise
366     * @param shift true if shift is part of this combined keypress, false otherwise
367     * @return an int that contains the information to represent the key with any modifiers as one value
368     */
369    public static int combineModifiers(char key, boolean alt, boolean ctrl, boolean shift) {
370        int c = alt ? (key | 0x10000) : key;
371        c |= ctrl ? 0x20000 : 0;
372        c |= shift ? 0x40000 : 0;
373        return c;
374    }

376    /**
377     * Gets the current key remapping as a String, which can be saved in a file and read back with
378     * {@link #keyMappingFromString(String)}. Because that method is automatically called by SquidInput's constructor
379     * and tries to load a file named "keymap.preferences" in the local folder, it is recommended that you save
380     * user-remapped keys to that file, but only if the user has actively changed them (that file could exist from an
381     * earlier run and you don't want to affect it unless the user made some change). This will allow input with any of
382     * Shift, Alt, and Ctl modifiers in any script supported by the Unicode Basic Multilingual Plane (the first 65536
383     * chars of Unicode).
384     * @return a String that stores four chars per remapping.
385     */
386    public String keyMappingToString()
387    {
388        char[] cs = new char[mapping.size << 2];
389        int i = 0;
390        for(IntIntMap.Entry ent : mapping)
391        {
392            cs[i++] = (char)(ent.key);
393            cs[i++] = (char)((ent.key>>>16) + 64);
394            cs[i++] = (char)(ent.value);
395            cs[i++] = (char)((ent.value>>>16) + 64);
396        }
397        return String.valueOf(cs);
398    }

400    /**
401     * Reads in a String (almost certainly produced by {@link #keyMappingToString()}) to set the current key remapping.
402     * This is automatically called if a file named "keymap.preferences" is present in the local folder, receiving the
403     * contents of that file as a parameter, so games that want to save a user's preferences should try to save that
404     * file if possible (this also allows one file to be reused across multiple computers or installations by copying
405     * it). This will allow input with any of Shift, Alt, and Ctl modifiers in any script supported by the Unicode Basic
406     * Multilingual Plane (the first 65536 chars of Unicode). You may want to call {@link #clearMapping()} before
407     * calling this if you have already set the mapping and want the String's contents to be used as the only mapping.
408     * @param keymap a String that was probably produced by {@link #keyMappingToString()}
409     * @return this for chaining
410     */
411    public SquidInput keyMappingFromString(String keymap)
412    {
413        int len, k, v;
414        if(keymap == null || (len = keymap.length()) < 4)
415            return this;
416        for (int i = 0; i < len - 3; i++) {
417            k = keymap.charAt(i);
418            k |= (keymap.charAt(++i) - 64) << 16;
419            v = keymap.charAt(++i);
420            v |= (keymap.charAt(++i) - 64) << 16;
421            mapping.put(k, v);
422        }
423        return this;
424    }

426    /**
427     * Processes all events queued up, passing them through this object's key processing and then to keyHandler. Mouse
428     * events are not queued and are processed when they come in.
429     */
430    public void drain () {
431        IntVLA qu = queue;
432
433        if (keyAction == null || qu.size <= 0) {
434            qu.clear();
435            return;
436        }
437
438        for (int i = 0, n = qu.size, t; i < n; ) {
439            t = qu.get(i++);
440            t = mapping.get(t, t);
441            keyAction.handle((char)t, (t & 0x10000) != 0, (t & 0x20000) != 0, (t & 0x40000) != 0);
442        }
443
444        qu.clear();
445    }

463    /**
464     * Returns true if at least one event is queued, but also will call {@link #keyDown(int)} if a key is being held but
465     * there is a failure to process the repeated event. The conditions this checks:
466     * <ul>
467     *     <li>Is a key is currently being held?</li>
468     *     <li>Are no modifier keys being held (shift, control, alt)? (without this check, the modifier key counts as a
469     *     repeat of the key it modifies, which is probably never the intended behavior)</li>
470     *     <li>Has {@link #keyDown(int)} already been called at least once?</li>
471     *     <li>Have there been at least {@link #getRepeatGap()} milliseconds between the last key being received and this call?</li>
472     * </ul>
473     * If all of these conditions are true, keyDown() is called again with the last key it had received, and if this has
474     * an effect (the key can be handled), then generally an event should be queued, so this will have a next event and
475     * should return true. You can change the amount of time required to hold a key to cause key repeats with
476     * {@link #setRepeatGap(long)}, but the default of 220 ms is usually suitable. Too low of a value can cause normal
477     * key presses to be counted twice or more, and too high of a value may delay an expected key repeat.
478     * @return true if there is an event queued, false otherwise
479     */
480    public boolean hasNext()
481    {
482        if(Gdx.input.isKeyPressed(Input.Keys.ANY_KEY)
483                && !Gdx.input.isKeyPressed(Input.Keys.SHIFT_LEFT)
484                && !Gdx.input.isKeyPressed(Input.Keys.SHIFT_RIGHT)
485                && !Gdx.input.isKeyPressed(Input.Keys.CONTROL_LEFT)
486                && !Gdx.input.isKeyPressed(Input.Keys.CONTROL_RIGHT)
487                && !Gdx.input.isKeyPressed(Input.Keys.ALT_LEFT)
488                && !Gdx.input.isKeyPressed(Input.Keys.ALT_RIGHT)
489                && lastKeyCode >= 0
490                && System.currentTimeMillis() - lastKeyTime > repeatGapMillis // defaults to 220 ms
491                )
492        {
493            keyDown(lastKeyCode);
494        }
495        return queue.size > 0;
496    }

498    /**
499     * Processes the first key event queued up, passing it to this object's InputProcessor. Mouse events are not
500     * queued and are processed when they come in.
501     */
502    public void next() {
503        IntVLA qu = queue;
504        if (keyAction == null) {
505            qu.clear(); // Accidentally captured a keypress when keys aren't meant to be handled
506            return;
507        }
508        if (qu.isEmpty()) {
509            return;
510        }
511        int t = qu.removeIndex(0);
512        t = mapping.get(t, t);
513        keyAction.handle((char)t, (t & 0x10000) != 0, (t & 0x20000) != 0, (t & 0x40000) != 0);
514    }

517     * Empties the backing queue of data.
518     */
519    public void flush()
520    {
521        queue.clear();
522    }

524    @Override
525        public boolean keyDown (int keycode) {
526        if (ignoreInput || keyAction == null) {
527            return false;
528        }
529        lastKeyTime = System.currentTimeMillis();
530        lastKeyCode = keycode;
531        boolean shift = Gdx.input.isKeyPressed(Input.Keys.SHIFT_LEFT) || Gdx.input.isKeyPressed(Input.Keys.SHIFT_RIGHT);
532        int c = fromCode(keycode, shift);
533        if(c != '\0') {
534            c |= (Gdx.input.isKeyPressed(Input.Keys.ALT_LEFT) || Gdx.input.isKeyPressed(Input.Keys.ALT_RIGHT))
535                    ? 0x10000 : 0;
536            c |= (Gdx.input.isKeyPressed(Input.Keys.CONTROL_LEFT) || Gdx.input.isKeyPressed(Input.Keys.CONTROL_RIGHT))
537                    ? 0x20000 : 0;
538            c |= (shift)
539                    ? 0x40000 : 0;
540            queue.add(c);
541        }
542        return false;
543    }

587    /**
588     * Maps keycodes to unicode chars, sometimes depending on whether the Shift key is held.
589     *
590     * It is strongly recommended that you refer to key combinations regarding non-alphabet keys by using, for example,
591     * Ctrl-Shift-; instead of Ctrl-:, that is to use the unshifted key with Shift instead of assuming that all
592     * keyboards will use the QWERTY layout. Pressing shift while pressing just about any representable symbol will map
593     * to the shifted version as if on a QWERTY keyboard, and if you don't have a QWERTY keyboard, the mappings are
594     * documented in full below.
595     *
596     * Keys 'a' to 'z' report 'A' to 'Z' when shift is held. Non-ASCII-Latin characters do not have this behavior, since
597     * most keyboards would be unable to send keys for a particular language and A-Z are very common. You can still
598     * allow a response to, e.g. '' and '' separately by checking for whether Shift was pressed in conjunction with
599     * '' on a keyboard with that key, which can be useful when users can configure their own keyboard layouts.
600     *
601     * Top row numbers map as follows:
602     *
603     * {@literal '1' to '!', '2' to '@', '3' to '#', '4' to '$', '5' to '%',}
604     * {@literal '6' to '^', '7' to '&amp;', '8' to '*', '9' to '(', '0' to ')'}
605     *
606     * Numpad numbers will report a SquidInput constant such as UP_LEFT_ARROW for Numpad 7, but only if numpadDirections
607     * is true; otherwise they send the number (here, 7). Numpad 0 sends VERTICAL_ARROW or 0.
608     *
609     * Most symbol keys are mapped to a single unicode char as a constant in SquidInput and disregard Shift. The
610     * constant is usually the same as the name of the char; possible exceptions are Backspace (on PC) or Delete (on
611     * Mac) mapping to BACKSPACE, Delete (on PC) mapping to FORWARD_DELETE, Esc mapping to ESCAPE, and Enter (on PC) or
612     * Return (on Mac) mapping to ENTER.
613     *
614     * {@literal ':', '*', '#', '@'}, and space keys, if present, always map to themselves, regardless of Shift.
615     *
616     * Other characters map as follows when Shift is held, as they would on a QWERTY keyboard:
617     * <ul>
618     * <li>{@code ','} to {@code '<'}</li>
619     * <li>{@code '.'} to {@code '>'}</li>
620     * <li>{@code '/'} to {@code '?'}</li>
621     * <li>{@code ';'} to {@code ':'}</li>
622     * <li>{@code '\''} to {@code '"'}</li>
623     * <li>{@code '['} to <code>'{'</code></li>
624     * <li>{@code ']'} to <code>'}'</code></li>
625     * <li>{@code '|'} to {@code '\\'}</li>
626     * <li>{@code '-'} to {@code '_'}</li>

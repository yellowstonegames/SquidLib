009/**
010 * This input processing class can handle mouse and keyboard input, using a squidpony.squidgrid.gui.gdx.SquidMouse for
011 * Mouse input and a user implementation of the SquidInput.KeyHandler interface to react to keys represented as chars
012 * and the modifiers those keys were pressed with, any of alt, ctrl, and/or shift. Not all keys are representable by
013 * default in unicode, so symbolic representations are stored in constants in this class, and are passed to
014 * KeyHandler.handle() as chars like DOWN_ARROW or its value, '\u2193'. Shift modifies the input as it would on a
015 * QWERTY keyboard, and the exact mapping is documented in fromKey() as well. This class handles mouse input
016 * immediately, but stores keypresses in a queue, storing all key events and allowing them to be processed one at a time
017 * using next() or all at once using drain(). To have an effect, it needs to be registered by calling
018 * Input.setInputProcessor(SquidInput).
019 * <br>
020 * This also allows some key remapping, including remapping so a key pressed with modifiers like Ctrl and Shift could
021 * act like '?' (which could be used by expert players to avoid accidentally opening a help menu they don't need), and
022 * that would free up '?' for some other use that could also be remapped. The remap() methods do much of this, often
023 * with help from {@link #combineModifiers(char, boolean, boolean, boolean)}, while the unmap() methods allow removal of
024 * any no-longer-wanted remappings.
025 * <br>
026 * It does not perform the blocking functionality of earlier SquidKey implementations, because this is meant to run
027 * in an event-driven libGDX game and should not step on the toes of libGDX's input handling. To block game logic
028 * until an event has been received, check hasNext() in the game's render() method and effectively "block" by not
029 * running game logic if hasNext() returns false. You can process an event if hasNext() returns true by calling next().
030 * Mouse inputs do not affect hasNext(), and next() will process only key pressed events.
031 *
032 * @author Eben Howard - http://squidpony.com - howard@squidpony.com
033 * @author Nathan Sweet
034 * @author Tommy Ettinger
035 * */
036public class SquidInput extends InputAdapter {
037    /**
038     * A single-method interface used to process "typed" characters, special characters produced by unusual keys, and
039     * modifiers that can affect them. SquidInput has numerous static char values that are expected to be passed
040     * to handle() in place of the special keys (such as arrow keys) that do not have a standard char value.
041     */
042    public interface KeyHandler{
043        /**
044         * The only method you need to implement yourself in KeyHandler, this should react to keys such as
045         * 'a' (produced by pressing the A key while not holding Shift), 'E' (produced by pressing the E key while
046         * holding Shift), and '\u2190' (left arrow in unicode, also available as a constant in SquidInput, produced by
047         * pressing the left arrow key even though that key does not have a default unicode representation). Capital
048         * letters will be capitalized when they are passed to this, but they may or may not have the shift argument as
049         * true depending on how this method was called. Symbols that may be produced by holding Shift and pressing a
050         * number or a symbol key can vary between keyboards (some may require Shift to be held down, others may not).
051         * <br>
052         * This can react to the input in whatever way you find appropriate for your game.
053         * @param key a char of the "typed" representation of the key, such as 'a' or 'E', or if there is no Unicode
054         *            character for the key, an appropriate alternate character as documented in SquidInput.fromKey()
055         * @param alt true if the Alt modifier was being held while this key was entered, false otherwise.
056         * @param ctrl true if the Ctrl modifier was being held while this key was entered, false otherwise.
057         * @param shift true if the Shift modifier was being held while this key was entered, false otherwise.
058         */
059        void handle(char key, boolean alt, boolean ctrl, boolean shift);
060    }

197    /**
198     * Remaps a char that could be input and processed by {@link KeyHandler#handle(char, boolean, boolean, boolean)}
199     * (possibly with some pressed modifiers) to another char with possible modifiers. When the first key/modifier mix
200     * is received, it will be translated to the second group in this method.
201     * @param pressedChar a source char that might be used in handling, like 'q', 'A', '7', '(', or {@link #UP_ARROW}.
202     * @param pressedAlt true if alt is part of the source combined keypress, false otherwise
203     * @param pressedCtrl true if ctrl is part of the source combined keypress, false otherwise
204     * @param pressedShift true if shift is part of the source combined keypress, false otherwise
205     * @param targetChar a target char that might be used in handling, like 'q', 'A', '7', '(', or {@link #UP_ARROW}.
206     * @param targetAlt true if alt is part of the target combined keypress, false otherwise
207     * @param targetCtrl true if ctrl is part of the target combined keypress, false otherwise
208     * @param targetShift true if shift is part of the target combined keypress, false otherwise
209     * @return this for chaining
210     */
211    public SquidInput remap(char pressedChar, boolean pressedAlt, boolean pressedCtrl, boolean pressedShift,
212                            char targetChar, boolean targetAlt, boolean targetCtrl, boolean targetShift)
213    {
214        mapping.put(combineModifiers(pressedChar, pressedAlt, pressedCtrl, pressedShift),
215                combineModifiers(targetChar, targetAlt, targetCtrl, targetShift));
216        return this;
217    }

219    /**
220     * Remaps a keypress combination, which is a char and several potential modifiers, to another keypress combination.
221     * When the {@code pressed} combination is received, it will be translated to {@code target} in this method.
222     * @see #combineModifiers(char, boolean, boolean, boolean) combineModifiers is usually used to make pressed and target
223     * @param pressed an int for the source keypress, probably produced by {@link #combineModifiers(char, boolean, boolean, boolean)}
224     * @param target an int for the target keypress, probably produced by {@link #combineModifiers(char, boolean, boolean, boolean)}
225     * @return this for chaining
226     */
227    public SquidInput remap(int pressed, int target)
228    {
229        mapping.put(pressed, target);
230        return this;
231    }
232    /**
233     * Remaps many keypress combinations, each of which is a char and several potential modifiers, to other keypress
234     * combinations. When the first of a pair of combinations is received, it will be translated to the second
235     * combination of the pair.
236     * @see #combineModifiers(char, boolean, boolean, boolean) combineModifiers is usually used to make the contents of pairs
237     * @param pairs an int array alternating source and target keypresses, each probably produced by {@link #combineModifiers(char, boolean, boolean, boolean)}
238     * @return this for chaining
239     */
240    public SquidInput remap(int[] pairs)
241    {
242        int len;
243        if(pairs == null || (len = pairs.length) <= 1)
244            return this;
245        for (int i = 0; i < len - 1; i++) {
246            mapping.put(pairs[i], pairs[++i]);
247        }
248        return this;
249    }

286    /**
287     * Combines the key (as it would be given to {@link KeyHandler#handle(char, boolean, boolean, boolean)}) with the
288     * three booleans for the alt, ctrl, and shift modifier keys, returning an int that can be used with the internal
289     * queue of ints or the public {@link #mapping} of received inputs to actual inputs the program can process.
290     * @param key a char that might be used in handling, like 'q', 'A', '7', '(', or {@link #UP_ARROW}.
291     * @param alt true if alt is part of this combined keypress, false otherwise
292     * @param ctrl true if ctrl is part of this combined keypress, false otherwise
293     * @param shift true if shift is part of this combined keypress, false otherwise
294     * @return an int that contains the information to represent the key with any modifiers as one value
295     */
296    public static int combineModifiers(char key, boolean alt, boolean ctrl, boolean shift) {
297        int c = alt ? (key | 0x10000) : key;
298        c |= ctrl ? 0x20000 : 0;
299        c |= shift ? 0x40000 : 0;
300        return c;
301    }

303    /**
304     * Gets the current key remapping as a String, which can be saved in a file and read back with
305     * {@link #keyMappingFromString(String)}. This will allow input with any of Shift, Alt, and Ctl modifiers in any
306     * script supported by the Unicode Basic Multilingual Plane (the first 65536 chars of Unicode).
307     * @return a String that stores four chars per remapping.
308     */
309    public String keyMappingToString()
310    {
311        char[] cs = new char[mapping.size << 2];
312        int i = 0;
313        for(IntIntMap.Entry ent : mapping)
314        {
315            cs[i++] = (char)(ent.key);
316            cs[i++] = (char)((ent.key>>>16) + 64);
317            cs[i++] = (char)(ent.value);
318            cs[i++] = (char)((ent.value>>>16) + 64);
319        }
320        return String.valueOf(cs);
321    }

323    /**
324     * Reads in a String (almost certainly produced by {@link #keyMappingToString()}) to set the current key remapping.
325     * This will allow input with any of Shift, Alt, and Ctl modifiers in any script supported by the Unicode Basic
326     * Multilingual Plane (the first 65536 chars of Unicode). You may want to call {@link #clearMapping()} before
327     * calling this if you have already set the mapping and want the String's contents to be used as the only mapping.
328     * @param keymap a String that was probably produced by {@link #keyMappingToString()}
329     * @return this for chaining
330     */
331    public SquidInput keyMappingFromString(String keymap)
332    {
333        int len, k, v;
334        if(keymap == null || (len = keymap.length()) < 4)
335            return this;
336        for (int i = 0; i < len - 3; i++) {
337            k = keymap.charAt(i);
338            k |= (keymap.charAt(++i) - 64) << 16;
339            v = keymap.charAt(++i);
340            v |= (keymap.charAt(++i) - 64) << 16;
341            mapping.put(k, v);
342        }
343        return this;
344    }

346    /**
347     * Processes all events queued up, passing them through this object's key processing and then to keyHandler. Mouse
348     * events are not queued and are processed when they come in.
349     */
350    public void drain () {
351        IntVLA qu = queue;
352
353        if (keyAction == null || qu.size <= 0) {
354            qu.clear();
355            return;
356        }
357
358        for (int i = 0, n = qu.size, t; i < n; ) {
359            t = qu.get(i++);
360            t = mapping.get(t, t);
361            keyAction.handle((char)t, (t & 0x10000) != 0, (t & 0x20000) != 0, (t & 0x40000) != 0);
362        }
363
364        qu.clear();
365    }

376    /**
377     * Processes the first key event queued up, passing it to this object's InputProcessor. Mouse events are not
378     * queued and are processed when they come in.
379     */
380    public void next() {
381        IntVLA qu = queue;
382        if (keyAction == null) {
383            qu.clear(); // Accidentally captured a keypress when keys aren't meant to be handled
384            return;
385        }
386        if (qu.isEmpty()) {
387            return;
388        }
389        int t = qu.removeIndex(0);
390        t = mapping.get(t, t);
391        keyAction.handle((char)t, (t & 0x10000) != 0, (t & 0x20000) != 0, (t & 0x40000) != 0);
392    }

402    @Override
403        public boolean keyDown (int keycode) {
404        if (ignoreInput || keyAction == null) {
405            return false;
406        }
407        boolean
408                shift = Gdx.input.isKeyPressed(Input.Keys.SHIFT_LEFT) || Gdx.input.isKeyPressed(Input.Keys.SHIFT_RIGHT);
409        int c = fromCode(keycode, shift);
410        if(c != '\0') {
411            c |= (Gdx.input.isKeyPressed(Input.Keys.ALT_LEFT) || Gdx.input.isKeyPressed(Input.Keys.ALT_RIGHT))
412                    ? 0x10000 : 0;
413            c |= (Gdx.input.isKeyPressed(Input.Keys.CONTROL_LEFT) || Gdx.input.isKeyPressed(Input.Keys.CONTROL_RIGHT))
414                    ? 0x20000 : 0;
415            c |= (shift)
416                    ? 0x40000 : 0;
417            queue.add(c);
418        }
419        return false;
420    }

464    /**
465     * Maps keycodes to unicode chars, sometimes depending on whether the Shift key is held.
466     *
467     * It is strongly recommended that you refer to key combinations regarding non-alphabet keys by using, for example,
468     * Ctrl-Shift-; instead of Ctrl-:, that is to use the unshifted key with Shift instead of assuming that all
469     * keyboards will use the QWERTY layout. Pressing shift while pressing just about any representable symbol will map
470     * to the shifted version as if on a QWERTY keyboard, and if you don't have a QWERTY keyboard, the mappings are
471     * documented in full below.
472     *
473     * Keys 'a' to 'z' report 'A' to 'Z' when shift is held. Non-ASCII-Latin characters do not have this behavior, since
474     * most keyboards would be unable to send keys for a particular language and A-Z are very common. You can still
475     * allow a response to, e.g. '' and '' separately by checking for whether Shift was pressed in conjunction with
476     * '' on a keyboard with that key, which can be useful when users can configure their own keyboard layouts.
477     *
478     * Top row numbers map as follows:
479     *
480     * {@literal '1' to '!', '2' to '@', '3' to '#', '4' to '$', '5' to '%',}
481     * {@literal '6' to '^', '7' to '&amp;', '8' to '*', '9' to '(', '0' to ')'}
482     *
483     * Numpad numbers will report a SquidInput constant such as UP_LEFT_ARROW for Numpad 7, but only if numpadDirections
484     * is true; otherwise they send the number (here, 7). Numpad 0 sends VERTICAL_ARROW or 0.
485     *
486     * Most symbol keys are mapped to a single unicode char as a constant in SquidInput and disregard Shift. The
487     * constant is usually the same as the name of the char; possible exceptions are Backspace (on PC) or Delete (on
488     * Mac) mapping to BACKSPACE, Delete (on PC) mapping to FORWARD_DELETE, Esc mapping to ESCAPE, and Enter (on PC) or
489     * Return (on Mac) mapping to ENTER.
490     *
491     * {@literal ':', '*', '#', '@'}, and space keys, if present, always map to themselves, regardless of Shift.
492     *
493     * Other characters map as follows when Shift is held, as they would on a QWERTY keyboard:
494     * <ul>
495     * <li>{@code ','} to {@code '<'}</li>
496     * <li>{@code '.'} to {@code '>'}</li>
497     * <li>{@code '/'} to {@code '?'}</li>
498     * <li>{@code ';'} to {@code ':'}</li>
499     * <li>{@code '\''} to {@code '"'}</li>
500     * <li>{@code '['} to <code>'{'</code></li>
501     * <li>{@code ']'} to <code>'}'</code></li>
502     * <li>{@code '|'} to {@code '\\'}</li>
503     * <li>{@code '-'} to {@code '_'}</li>
504     * <li>{@code '+'} to {@code '='}</li>
505     * <li>{@code '`'} to {@code '~'} (note, this key produces no event on the GWT backend)</li>
506     * </ul>
507     * @param keycode a keycode as passed by LibGDX
508     * @param shift true if Shift key is being held.
509     * @return a char appropriate to the given keycode; often uses shift to capitalize or change a char, but not for keys like the arrow keys that normally don't produce chars
510     */
511    public char fromCode(int keycode, boolean shift)
512    {
513        switch (keycode) {
514            case Input.Keys.HOME:
515                return HOME;
516            case Input.Keys.FORWARD_DEL:
517                return FORWARD_DELETE;
518            case Input.Keys.ESCAPE:
519                return ESCAPE;
520            case Input.Keys.END:
521                return END;
522
523            case Input.Keys.UP:
524                return UP_ARROW;
525            case Input.Keys.DOWN:
526                return DOWN_ARROW;
527            case Input.Keys.LEFT:
528                return LEFT_ARROW;
529            case Input.Keys.RIGHT:
530                return RIGHT_ARROW;
531            case Input.Keys.CENTER:
532                return CENTER_ARROW;
533
534            case Input.Keys.NUM_0:
535                return (shift) ? ')' : '0';
